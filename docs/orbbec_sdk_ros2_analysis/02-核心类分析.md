## 2. OBCameraNodeDriver 类详解

### 2.1 角色定位
继承自 `rclcpp::Node`，是 ROS2 系统中唯一的"真正节点"，职责：
- 设备发现、选择、连接
- 热插拔处理
- 创建和管理 OBCameraNode 或 OBLidarNode
- 设备重启和固件更新
- 设备状态监控与发布

### 2.2 核心成员变量

```cpp
// SDK 上下文——全局唯一，管理所有设备
std::unique_ptr<ob::Context> ctx_;

// 设备对象
std::shared_ptr<ob::Device>     device_;
std::shared_ptr<ob::DeviceInfo> device_info_;

// 业务节点——根据设备类型创建其一
std::unique_ptr<OBCameraNode>            ob_camera_node_;
std::unique_ptr<orbbec_lidar::OBLidarNode> ob_lidar_node_;

// 设备选择参数
std::string serial_number_;   // 按序列号
std::string usb_port_;        // 按 USB 端口
std::string net_device_ip_;   // 按网络 IP

// 连接状态
std::atomic_bool device_connected_{false};
std::atomic_bool device_connecting_{false};

// 进程间同步锁（共享内存）
pthread_mutex_t* orb_device_lock_;

// 设备状态发布
rclcpp::Publisher<orbbec_camera_msgs::msg::DeviceStatus>::SharedPtr device_status_pub_;
```

### 2.3 初始化流程

```
OBCameraNodeDriver 构造函数
    │
    ▼
init()
    ├── 创建 ob::Context（SDK 上下文）
    ├── 设置日志级别
    ├── 注册设备变更回调（onDeviceConnected / onDeviceDisconnected）
    ├── 加载参数（序列号、USB 端口、网络 IP 等）
    ├── 创建共享内存进程锁
    ├── 连接网络设备（如果配置了 net_device_ip）
    ├── 启动设备发现定时器（checkConnectTimer）
    ├── 启动设备状态发布定时器（deviceStatusTimer）
    └── 应用 Force IP 配置（如果启用）
```

### 2.4 设备选择逻辑

```cpp
// selectDevice() 选择策略优先级：
// 1. serial_number 非空 → selectDeviceBySerialNumber()
// 2. usb_port 非空      → selectDeviceByUSBPort()
// 3. net_device_ip 非空 → selectDeviceByNetIP()
// 4. 均为空             → 选择 DeviceList 中第一个设备
```

### 2.5 热插拔处理

```
设备插入 → onDeviceConnected()  → selectDevice() + initializeDevice()
设备拔出 → onDeviceDisconnected() → 清理 OBCameraNode/OBLidarNode → 等待重连
```

## 3. OBCameraNode 类详解

### 3.1 角色定位
不继承 `rclcpp::Node`，接收 Node 指针作为构造参数，是深度相机的核心业务逻辑类。

### 3.2 构造函数签名

```cpp
OBCameraNode(rclcpp::Node* node,
             std::shared_ptr<ob::Device> device,
             std::shared_ptr<Parameters> parameters,
             bool use_intra_process = false);
```

### 3.3 流类型定义（stream_index_pair）

`stream_index_pair = std::pair<ob_stream_type, int>`

| 流索引常量 | SDK 类型 | 说明 |
|-----------|---------|------|
| COLOR      | OB_STREAM_COLOR       | 彩色相机 |
| COLOR_LEFT | OB_STREAM_COLOR_LEFT  | 左彩色（双目设备） |
| COLOR_RIGHT| OB_STREAM_COLOR_RIGHT | 右彩色（双目设备） |
| DEPTH      | OB_STREAM_DEPTH       | 深度流 |
| INFRA0     | OB_STREAM_IR          | IR 红外流 |
| INFRA1     | OB_STREAM_IR_LEFT     | 左 IR 流 |
| INFRA2     | OB_STREAM_IR_RIGHT    | 右 IR 流 |
| GYRO       | OB_STREAM_GYRO        | 陀螺仪 |
| ACCEL      | OB_STREAM_ACCEL       | 加速度计 |

### 3.4 核心数据结构

```cpp
// 按 stream_index_pair 组织的映射表
std::map<stream_index_pair, std::shared_ptr<ob::Sensor>>          sensors_;
std::map<stream_index_pair, sensor_msgs::msg::CameraInfo>         camera_infos_;
std::map<stream_index_pair, int>                                  width_, height_, fps_;
std::map<stream_index_pair, ob_format>                            format_;
std::map<stream_index_pair, bool>                                 enable_stream_;
std::map<stream_index_pair, std::shared_ptr<ob::StreamProfile>>   stream_profile_;
std::map<stream_index_pair, std::shared_ptr<image_publisher>>     image_publishers_;
```

### 3.5 初始化流程

```
OBCameraNode 构造函数
    │
    ├── setupDevices()              # 枚举传感器
    ├── setupDefaultImageFormat()   # 设置默认图像格式
    ├── getParameters()             # 从参数服务器加载配置
    ├── setupProfiles()             # 匹配最佳流配置
    ├── setupTopics()               # 创建 Topic / Publisher
    │   ├── setupPublishers()       # 图像 / 点云发布者
    │   ├── setupCameraInfo()       # 相机内参信息
    │   └── setupCameraCtrlServices() # 控制服务
    ├── setupPipelineConfig()       # 配置 SDK Pipeline
    ├── setupDiagnosticUpdater()    # 诊断更新器
    └── publishStaticTransforms()   # 发布静态 TF
```

### 3.6 帧处理管线

```
ob::Pipeline 回调
    │
    ▼
onNewFrameSetCallback(FrameSet)
    ├── Color  Frame → softwareDecodeColorFrame() → onNewFrameCallback(COLOR)
    ├── Depth  Frame → processDepthFrameFilter()  → onNewFrameCallback(DEPTH)
    ├── IR Left Frame → processLeftIrFrameFilter()  → onNewFrameCallback(INFRA1)
    ├── IR Right Frame → processRightIrFrameFilter() → onNewFrameCallback(INFRA2)
    └── publishPointCloud(frame_set)

onNewFrameCallback(frame, stream_index)
    ├── getFrameTimestampUs()         # 时间戳
    ├── 格式转换 / 翻转 / 镜像 / 旋转
    ├── 构建 sensor_msgs::msg::Image
    ├── 构建 sensor_msgs::msg::CameraInfo
    ├── image_publishers_[sip]->publish()
    └── 保存图像（如果触发）
```

### 3.7 关键内部结构——IMUData

```cpp
struct IMUData {
    stream_index_pair stream_;   // GYRO 或 ACCEL
    Eigen::Vector3d   data_;     // 3 轴数据
    double            timestamp_; // 纳秒时间戳
};
```

### 3.8 线程安全设计

| 锁 / 原子量 | 保护对象 |
|------------|---------|
| `device_lock_` (recursive_mutex) | 设备操作 |
| `point_cloud_mutex_` (recursive_mutex) | 点云生成 |
| `color_frame_queue_lock_` + cv | 颜色帧异步队列 |
| `cleaning_` (atomic_bool) | 防止重复清理 |
| `withDeviceLock()` 模板方法 | 对外暴露安全锁访问 |

## 4. OBLidarNode 类详解

### 4.1 核心功能
- 发布 `sensor_msgs::msg::LaserScan`（2D 激光扫描）
- 发布 `sensor_msgs::msg::PointCloud2`（3D 点云，笛卡尔 / 球面）
- 多帧累积（`enable_cloud_accumulated`）
- 帧过滤与角度范围裁剪
- IMU 数据处理

### 4.2 LiDAR 特有参数

```cpp
float min_angle_  = -135.0;  // 最小扫描角度 (°)
float max_angle_  =  135.0;  // 最大扫描角度 (°)
float min_range_  =  0.05;   // 最小测距范围 (m)
float max_range_  =  30.0;   // 最大测距范围 (m)
int   publish_n_pkts_ = 1;   // 多帧发布
bool  enable_cloud_accumulated_ = false; // 点云累积
```

## 5. Parameters 类（动态参数管理）

### 5.1 设计模式

```
Parameters → ParametersBackend → rclcpp::Node
                                      │
                                      ▼
                              ROS2 参数服务器
                         （支持 ros2 param set/get）
```

### 5.2 主要方法

| 方法 | 说明 |
|------|------|
| `setParam()` | 声明参数并注册变更回调 |
| `setParamT()` | 泛型版本，直接绑定成员变量 |
| `setParamValue()` | 运行时更新参数值（线程安全） |
| `removeParam()` | 删除参数 |
