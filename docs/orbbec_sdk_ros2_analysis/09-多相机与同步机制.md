# 多相机与同步机制分析

## 1. 多相机部署架构

### 1.1 部署方式

```
方式 1：多进程（推荐）
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ Process 1    │  │ Process 2    │  │ Process 3    │
│ camera_1     │  │ camera_2     │  │ camera_3     │
│ (SN: AAA)    │  │ (SN: BBB)    │  │ (SN: CCC)    │
└──────────────┘  └──────────────┘  └──────────────┘
       │                 │                  │
       └─────────────────┼──────────────────┘
                        USB

方式 2：组合容器（低延迟，intra-process 通信）
┌─────────────────────────────────────────────┐
│ ComposableNodeContainer                      │
│  ├── OBCameraNodeDriver（camera_1）          │
│  ├── OBCameraNodeDriver（camera_2）          │
│  └── OBCameraNodeDriver（camera_3）          │
└─────────────────────────────────────────────┘
```

### 1.2 Launch 文件多相机配置

```python
for cam in [{'name':'camera_1','serial':'AAA'},
            {'name':'camera_2','serial':'BBB'}]:
    nodes.append(
        IncludeLaunchDescription(
            PythonLaunchDescriptionSource('gemini_330_series.launch.py'),
            launch_arguments=cam.items(),
        )
    )
```

## 2. 硬件同步模式

### 2.1 OBMultiDeviceSyncMode

| 模式 | 说明 |
|------|------|
| FREE_RUN | 自由运行（默认） |
| STANDALONE | 独立模式 |
| PRIMARY | 主设备（提供触发信号） |
| SECONDARY | 从设备（接收触发信号） |
| SECONDARY_SYNCED | 同步从设备 |
| SOFTWARE_TRIGGERING | 软件触发模式 |
| HARDWARE_TRIGGERING | 硬件触发模式 |

### 2.2 同步参数

| 参数 | 说明 |
|------|------|
| depth_delay_us | 深度帧触发延迟（μs） |
| color_delay_us | 彩色帧触发延迟（μs） |
| trigger2image_delay_us | 触发到出图延迟 |
| trigger_out_delay_us | 触发输出延迟 |
| trigger_out_enabled | 触发输出使能 |
| frames_per_trigger | 每次触发帧数 |

### 2.3 主从同步配置示例

```yaml
# 主设备（Primary）
sync_mode: "primary"
trigger_out_enabled: true
trigger_out_delay_us: 0

# 从设备（Secondary）
sync_mode: "secondary"
depth_delay_us: 0
color_delay_us: 0
trigger2image_delay_us: 0
```

## 5. 帧同步

### 5.1 帧聚合模式

```cpp
// frame_aggregate_mode_：
"full_frame"  → 等待所有流都有帧后再回调（最严格）
```

## 6. GMSL 触发（Jetson 平台）

```cpp
// GMSL 相机通过 /dev/camsync 触发
int openSocSyncPwmTrigger(uint16_t fps) {
    gmsl_trigger_fd_ = open("/dev/camsync", O_RDWR);
    cs_param_t param = {.mode = 1, .fps = fps};
    ioctl(gmsl_trigger_fd_, SET_PARAM, &param);
}
void startGmslTrigger() { ioctl(gmsl_trigger_fd_, START_TRIGGER); }
void stopGmslTrigger()  { ioctl(gmsl_trigger_fd_, STOP_TRIGGER);  }
```

## 7. Interleave 模式（交错曝光）

### 7.1 HDR 交错

```
Frame Index 0：短曝光 + 高激光功率
Frame Index 1：长曝光 + 低激光功率
→ HDRMerge 滤波器合成高动态范围深度图
```

### 7.2 激光交错

```
Frame Index 0：激光开（投射结构光）
Frame Index 1：激光关（纯被动）
→ SequenceIdFilter 选择特定帧
```
