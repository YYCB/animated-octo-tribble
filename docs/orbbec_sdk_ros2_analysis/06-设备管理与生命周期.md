# 设备管理与生命周期分析

## 1. 设备发现机制

### 1.1 ob::Context 初始化

```cpp
ctx_ = std::make_unique<ob::Context>(config_path_);

// 注册设备变更回调
device_changed_callback_id_ = ctx_->registerDeviceChangedCallback(
    [this](std::shared_ptr<ob::DeviceList> removed,
           std::shared_ptr<ob::DeviceList> added) {
        onDeviceDisconnected(removed);
        onDeviceConnected(added);
    });
```

### 1.2 设备发现策略

```
Context 初始化
    ├── USB 设备自动枚举（默认）—— libuvc / v4l2 后端
    ├── 网络设备枚举（enumerate_net_device: true）
    │   └── connectNetDevice(ip, port)
    └── GMSL 设备（Jetson 平台）
```

### 1.3 设备选择优先级

```
selectDevice(DeviceList)
    ├── serial_number 非空  → selectDeviceBySerialNumber()
    ├── usb_port 非空       → selectDeviceByUSBPort()
    ├── net_device_ip 非空  → selectDeviceByNetIP()
    └── 均为空              → 选择 DeviceList 第一个设备
```

## 2. 设备生命周期状态机

```
[未连接]
    │  发现设备（onDeviceConnected）
    ▼
[连接中]  device_connecting_ = true
    │  selectDevice() + initializeDevice()
    ▼
[已初始化]  device_connected_ = true
    │  startStreams() / startIMU()
    ▼
[运行中]  pipeline_started_ = true
    │  发布图像 / 点云 / IMU；处理 Service；设备状态监控
    │  设备断开（onDeviceDisconnected）
    ▼
[断开中]  clean() / 释放资源
    │  checkConnectTimer()（每秒检查）
    ▼
[等待重连] ──────────────────────────────→ [未连接]
```

## 3. 设备初始化详细流程

```cpp
void initializeDevice(device) {
    // 1. 获取设备信息
    device_info_ = device->getDeviceInfo();
    auto pid  = device_info_->pid();

    // 2. 主机时间同步（可选）
    if (enable_sync_host_time_) device->timestampReset();

    // 3. 固件更新（可选）
    if (!upgrade_firmware_.empty())
        firmwareUpdate(upgrade_firmware_);

    // 4. 预设固件应用（可选）
    if (!preset_firmware_path_.empty())
        updatePresetFirmware(preset_firmware_path_);

    // 5. 设备类型判断与节点创建
    if (device_type == OB_STRUCTURED_LIGHT_MONOCULAR_CAMERA
        || device_type == OB_LIDAR) {
        ob_lidar_node_ = std::make_unique<OBLidarNode>(...);
    } else {
        ob_camera_node_ = std::make_unique<OBCameraNode>(...);
    }

    // 6. 启动数据流
    ob_camera_node_->startStreams();
    ob_camera_node_->startIMU();

    device_connected_ = true;
}
```

## 4. 热插拔机制

### 4.1 设备连接回调

```cpp
void onDeviceConnected(device_list) {
    if (!device_connected_ && !device_connecting_)
        startDevice(device_list);
}
```

### 4.2 设备断开回调

```cpp
void onDeviceDisconnected(device_list) {
    // 检查是否是当前设备
    // → device_connected_ = false
    // → 清理 OBCameraNode / OBLidarNode
    // → 释放 device_ 对象，等待重连
}
```

### 4.3 重连定时器

```cpp
void checkConnectTimer() {
    // 每秒执行一次
    if (!device_connected_ && !device_connecting_)
        queryDevice();
}
```

## 5. 进程间设备锁

### 5.1 共享内存锁机制

```cpp
orb_device_lock_shm_fd_ = shm_open(ORB_DEFAULT_LOCK_NAME, O_CREAT|O_RDWR, 0666);
ftruncate(orb_device_lock_shm_fd_, sizeof(pthread_mutex_t));
orb_device_lock_shm_addr_ = mmap(...);
orb_device_lock_ = (pthread_mutex_t*)orb_device_lock_shm_addr_;

pthread_mutexattr_setpshared(&orb_device_lock_attr_, PTHREAD_PROCESS_SHARED);
pthread_mutex_init(orb_device_lock_, &orb_device_lock_attr_);
```

### 5.2 使用场景
多相机部署时每个相机节点在独立进程中运行，进程锁确保同一时刻只有一个进程进行设备初始化。
