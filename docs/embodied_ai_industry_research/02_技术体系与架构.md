# 02 技术体系与架构

## 2.1 总体架构：三层栈

具身智能系统通常可以抽象为三层闭环：

```
┌──────────────────────────────────────────────────────┐
│  L3 任务/认知层  Cognition / Task Planning           │
│   - 目标理解、长程规划、人机对话                     │
│   - 代表：LLM / VLM / Agent Framework                │
│   - 频率：0.1 – 1 Hz                                 │
├──────────────────────────────────────────────────────┤
│  L2 技能/策略层  Skill / Policy                      │
│   - 短程动作策略：抓取、行走、开门、放置             │
│   - 代表：VLA、Diffusion Policy、ACT、RL Policy      │
│   - 频率：5 – 50 Hz                                  │
├──────────────────────────────────────────────────────┤
│  L1 控制/驱动层  Control / Actuation                 │
│   - 关节伺服、力控、WBC、MPC、安全限位               │
│   - 代表：Cascade PID / MPC / WBC / Impedance        │
│   - 频率：500 Hz – 2 kHz                             │
└──────────────────────────────────────────────────────┘
            ▲ 传感器（相机 / IMU / 力扭 / 触觉 / LiDAR）
            ▼ 执行器（电机 / 液压 / 灵巧手 / 抓夹）
```

不同公司在三层之间的"切分点"不同，决定了其技术风格：
- **传统派 + LLM 上盖**：L1/L2 用经典控制+规划，仅 L3 接 LLM。代表：早期 SayCan、波士顿动力 + Spot LLM 接口。
- **端到端 VLA**：L2/L3 合并为单一大模型，输入图像/语言，输出动作。代表：RT-2、π0、Helix（Figure）。
- **世界模型派**：核心是学习一个可预测的"世界模型"，再在其上规划。代表：V-JEPA、Genie、1X World Model、Dreamer 系。

## 2.2 主流技术范式

### 2.2.1 模块化 Pipeline（Classical Stack）

```
感知 → 状态估计 → 任务规划 → 运动规划 → 控制 → 执行
（CV/SLAM）（EKF/优化）（PDDL/HTN）（OMPL/CHOMP）（MPC/WBC）
```

- 优点：可解释、可调、单模块易优化；
- 缺点：模块间误差累积、长尾鲁棒性差、新增任务工程量大；
- 现状：仍是工业机器人和移动底盘主流；高 DOF 通用操作正快速被 L2 端到端取代。

### 2.2.2 模仿学习 Imitation Learning

通过人类示教或遥操作收集 (obs → action) 序列，监督学习一个策略。

| 方法 | 关键思想 | 代表工作 |
|------|---------|----------|
| Behavioral Cloning (BC) | 直接监督 action | 早期所有 IL |
| DAgger | 在线纠偏，缓解协变量漂移 | DAgger 2011 |
| ACT (Action Chunking Transformer) | 输出动作"块"，提高时序一致性 | Stanford ALOHA |
| Diffusion Policy | 用扩散模型建模多模态动作分布 | Toyota / CMU 2023 |
| 3D Diffusion Policy | 显式利用 3D 点云输入 | 2024 |

**优点**：数据 → 能力路径直接、可观测；
**缺点**：依赖大量人示教（昂贵）、泛化弱、对相机视角和初始条件敏感。

### 2.2.3 强化学习 RL（Sim-Centric）

在仿真器中大规模训练，再迁移到真机。

- 在四足/人形 **运动控制**（locomotion）上极为成功：宇树、ANYmal、Boston Dynamics 等大量使用 RL。
- 在 **操作** 上效果有限，原因是接触动力学和视觉差异更大。
- 关键技术：Domain Randomization、Privileged Learning、Teacher-Student、RMA、PPO/SAC。

### 2.2.4 VLA（Vision-Language-Action）

> 输入：摄像头图像 + 自然语言指令；输出：连续动作 / 离散 action token。

```
[图像]──┐
        ├──► VLM 主干 ──► Action Head ──► 7~50 维动作
[语言]──┘    （PaLI/Llama/Qwen-VL 等）   （token化或扩散）
```

代表：
- **RT-2 (Google, 2023)**：将 PaLI-X 的 VLM 与机器人动作 token 联合训练；
- **OpenVLA (2024 开源)**：基于 Llama-2 + Prismatic VLM 的开源 VLA，社区基线；
- **π0 / π0.5 (Physical Intelligence, 2024–2025)**：流匹配式动作头，跨本体训练；
- **GR00T N (NVIDIA)**：人形通用基础模型；
- **Helix (Figure, 2025)**：双系统，慢思考 VLM + 快控制策略；
- **国产**：智元 GO-1、银河通用 GroceryVLA、星动纪元 ERA-42、清华 RDT-1B、上海 AI Lab RoboBrain 等。

**关键创新**：
1. **跨本体训练**：在 Open X-Embodiment（22 个机器人 / 100+ 万条轨迹）上联合训练；
2. **动作 token 化** 或 **扩散 / 流匹配 head**：统一异构动作空间；
3. **共训互联网数据 + 机器人数据**：保留语义泛化能力。

### 2.2.5 世界模型 World Model

学习一个能从当前状态预测未来观测的模型 `p(o_{t+1:T} | o_{≤t}, a_{≤t})`，在其上规划或采样动作。

代表工作：
- **Dreamer V1/V2/V3**（DeepMind）：在隐空间想象然后 RL；
- **V-JEPA / V-JEPA 2**（Meta）：预测视觉表征，而非像素；
- **Genie / Genie 2 / 3**（DeepMind）：可交互的视频生成器；
- **1X World Model**（1X Technologies）：用视频生成做策略评估；
- **NVIDIA Cosmos**：物理一致的视频生成基础模型，专为具身和自驾；
- **Sora 等通用视频模型**：被探索作为隐式世界模型。

世界模型有两条路径：**生成像素**（Cosmos / Genie / Sora） vs **预测表征**（JEPA），后者计算更便宜、长期可控性更好；目前业界尚未收敛。

### 2.2.6 双系统架构（System-1 / System-2）

借鉴 Kahneman 思想，把"慢思考 + 快反应"分开：

| 系统 | 角色 | 算力 | 频率 | 模型形态 |
|------|------|------|------|----------|
| System-2 | 任务理解、长程规划 | 云 / 大算力 | < 5 Hz | 大型 VLM / LLM |
| System-1 | 反应式控制 | 端侧 | 10–500 Hz | 轻量 VLA / Diffusion / RL Policy |

代表：Figure Helix、谷歌 Gemini Robotics（2025）、银河通用 GraspVLA + Reactive policy 组合。

## 2.3 感知子系统

| 模态 | 主流传感器 | 关键算法 |
|------|-----------|---------|
| 视觉 RGB | 单目/双目工业相机、Realsense / Orbbec / ZED | VLM 视觉编码、6D Pose、Open-Vocabulary 检测 |
| 深度 | iToF / 结构光 / 双目 | 点云分割、Occupancy、NeRF / 3DGS 重建 |
| LiDAR | 机械式 / 固态 | 3D 检测、SLAM、避障 |
| 力 / 力矩 | 关节内置力矩传感、6 轴 FT | Impedance、力控插装 |
| 触觉 | GelSight、DIGIT、电容阵列 | 滑移检测、形状识别、灵巧操作 |
| 本体感觉 | 编码器、IMU、足底力 | 状态估计、Locomotion |
| 听觉 | 麦克风阵列 | 语音指令、声源定位 |

**趋势**：触觉感知爆发（Meta Sparsh、GelSight Mini、Tactile-VLA），是从"看得见"到"摸得准"的关键。

## 2.4 执行子系统

```
关节驱动选项：
 ├─ 谐波减速器 + 无框力矩电机     （高减速比、轻量、主流人形）
 ├─ 行星齿轮 + 伺服               （低成本、噪声大）
 ├─ 准直驱（Quasi-Direct Drive）  （MIT Mini Cheetah、宇树 Go/H1）
 ├─ 液压                          （Atlas 旧版、Sarcos）→ 维护贵，逐步退出
 └─ 直线推杆 / 串联弹性（SEA）    （部分人形腿部）
```

灵巧手：从工业二指夹爪 → 多指（5–22 DOF）腱驱动 / 直驱 / 气动；触觉 + 力控成主流方向（Shadow Hand、Tesla Hand、Inspire、星动纪元 XHand、PaXini DexH13）。

## 2.5 软件栈

```
应用层  │ 任务编排、UI、远端运维
────────┼──────────────────────────────────
中间件  │ ROS 2 / Cyclone DDS / Fast DDS / 自研 RT 框架
────────┼──────────────────────────────────
模型/推理│ TensorRT / ONNXRuntime / Triton / vLLM / TVM
────────┼──────────────────────────────────
控制器  │ EtherCAT / CANopen / 自研 RS485 / 实时内核 (PREEMPT_RT / Xenomai)
────────┼──────────────────────────────────
OS      │ Ubuntu + RT 补丁、QNX、Yocto 定制
────────┼──────────────────────────────────
硬件    │ 边缘 SoC（Jetson Orin/Thor、地平线 J6、寒武纪、自研）
```

> 本仓库 `chassis_protocol/` 即是一个面向移动底盘的 ROS 2 ament_cmake 驱动 HAL 包（proto → frame_codec → ITransport → ChassisHal → ROS2 ChassisNode）的真实示例，反映了具身智能"中间件 + 驱动 HAL"层的典型工程组织方式。

## 2.6 范式对比一表

| 维度 | 模块化 Pipeline | RL（Sim-Centric） | 模仿学习 IL | VLA 端到端 | 世界模型 |
|------|----------------|------------------|------------|-----------|---------|

| 主要瓶颈 | 长尾、扩展性 | Sim2Real | 数据规模 | 数据 + 推理 | 算力 + 长程一致性 |

## 2.7 小结

- 短期（1–2 年）：**VLA + 经典控制底层** 的混合架构是行业最务实的工程方案；
- 中期（3–5 年）：**双系统 + 世界模型** 有望成为通用机器人的"基础模型"形态；
- 长期：物理大模型 + 触觉等多模态可能孕育出"机器人版 GPT 时刻"，但仍受限于数据与本体多样性。
