# 04 核心技术栈

本章深入展开第 02 章涉及的关键技术，按"模型 / 学习 / 数据 / 仿真 / Sim2Real"五个维度。

## 4.1 模型篇：从 LLM 到 VLA / 世界模型

### 4.1.1 VLA 模型架构演化

```
RT-1 (2022)         模仿学习 + Transformer       仅图像/状态
        │
        ▼
RT-2 (2023)         VLM 主干 + 动作 token        引入语义泛化
        │
        ▼
OpenVLA (2024)      Llama-2 + Prismatic VLM      开源、社区基线
        │
        ▼
π0 / π0.5 (2024-25) 流匹配 (flow matching) 头     连续动作、跨本体
        │
        ▼
Helix / GR00T N / Gemini Robotics (2025)
                    双系统 / 多本体 / 触觉-VLA / 长程规划
```

关键设计选择：
| 设计点 | 选项 | 折中 |
|--------|-----|------|
| 视觉编码 | SigLIP / DINOv2 / CLIP / 自研 | 语义 vs 几何精度 |
| 语言主干 | Llama / Qwen / Gemma / 私有 | 开源生态 vs 推理速度 |
| 动作头 | token化 / MLP / Diffusion / Flow Matching | 离散表达 vs 平滑高频 |
| 上下文长度 | 当前帧 / 短视频 / 长 history | 长程任务能力 vs 算力 |
| 输出维度 | 单本体 / 跨本体共享 | 数据效率 vs 性能上限 |
| 推理频率 | 5–50 Hz | 用 Action Chunk 摊薄 |

### 4.1.2 世界模型路线

两条路径：

| 类型 | 代表 | 特点 |
|------|------|------|
| **像素生成** | Sora、Genie、NVIDIA Cosmos、1X World Model | 直观、可视化、用于数据合成与策略评估，但物理一致性差、推理慢 |
| **表征预测** | V-JEPA、V-JEPA 2、DINO-WM | 不重建像素，预测潜空间表征；轻量、长程稳定，但可视化弱 |

世界模型的实际产业用途（2025 视角）：
1. **数据增广**：基于真实视频合成新视角 / 新物体下的轨迹；
2. **策略评估 (offline eval)**：给定策略，在世界模型中跑评分，避免昂贵真机评估；
3. **想象式规划**：MPC 在隐空间展开，用于长程任务。

### 4.1.3 多模态扩展

新出现的模态：
- **触觉**：Meta Sparsh（统一触觉 SSL）、Tactile-VLA；
- **音频**：开门、撕胶带、倒水等音事件；
- **本体感觉**：力 / 力矩 / 关节速度作为额外 token；
- **3D 表示**：Voxel / Point / 3DGS / NeRF 输入。

## 4.2 学习篇：模仿、强化与混合

### 4.2.1 模仿学习的工程化

ALOHA / Mobile-ALOHA 范式：
1. 双臂遥操作主从结构，单任务采集 50–200 条；
2. 用 ACT / Diffusion Policy 监督学习；
3. 在受控初始条件下任务成功率可达 80–90%。

工业化趋势：
- 数据采集设备产品化：UMI（Universal Manipulation Interface 手持夹爪）、Mobile-ALOHA、Open-TeleVision、AnyTeleop、Apple Vision Pro 遥操作；
- 数据规模化：过去单任务 200 条 → 跨任务跨本体百万条。

### 4.2.2 强化学习的工业化

主要在 **腿足运动控制** 上完全成功：
- 训练范式：Isaac Gym / Isaac Lab 大规模并行 → Domain Randomization → Teacher-Student 蒸馏 → 真机；
- 关键技术：PPO + Asymmetric Actor-Critic、Privileged Critic、Adaptation Module（RMA）、Periodic Reward；
- 代表：MIT Cheetah、ANYmal、宇树 H1（行走 / 跑步 / 跳跃）；
- **操作上的 RL** 仍以 *fine-tune* 形式存在（在 IL 之上做 RL refinement）。

### 4.2.3 混合范式

实务上"先 IL 再 RL 微调"已是事实标准：
```
预训练 VLA（互联网 + 真机）→ 任务级 IL → RL 微调（仿真 + 少量真机） → 部署
```

新兴方向：
- **基于人类视频的预训练**（VPT、HumanPlus、AnyDoor）；
- **In-Context Imitation**（少样本即学会新任务）；
- **Foundation policy + Skill library**（基础策略 + 技能库分离）。

2. **照片级渲染**（Omniverse / GS based）大幅缩小视觉 sim2real gap；
3. **场景自动生成**（procedural generation, 例如 RoboCasa）解决场景多样性。

## 4.5 Sim2Real

主流缓解策略：
1. **Domain Randomization**：随机化质量、摩擦、纹理、光照、相机参数；
2. **Privileged / Teacher-Student**：仿真中用上帝信息训 teacher，真机用部分观测 student；
3. **System Identification**：参数化建模 + 优化拟合真机；
4. **Adaptation Module / RMA**：在线推断动力学参数；
5. **Real2Sim2Real**：用真机数据重建场景再训练；
6. **Co-training**：同时混合仿真和真机数据训练。

操作类任务 sim2real 仍是开放问题，主要难点在 **接触动力学** 和 **变形物体**。

## 4.6 推理与部署

| 阶段 | 模型规模 | 算力需求 | 备注 |
|------|---------|---------|------|
| L3 任务规划 LLM/VLM | 7B–70B | 云或大算力边缘 | 可远程，时延宽容 |
| L2 VLA 策略 | 1B–7B | 单 Jetson Thor / Orin AGX | 5–30 Hz 推理，常用 INT8/FP8 |
| L1 控制环 | < 100 K params 或经典控制 | MCU / RTOS | 500 Hz–2 kHz |

优化技术：量化（INT8/FP8/INT4）、Speculative Decoding、KV Cache 重用、Action Chunking、模型蒸馏。

## 4.7 安全与可解释

- **运行时安全**：ISO 13482（个人服务机器人）、ISO 10218（工业机器人）、IEC 61508 SIL；
- **行为限制**：safe RL、CBF（Control Barrier Functions）、力 / 速度限位；
- **解释性**：注意力可视化、动作不确定性估计、reasoning trace 输出（Gemini Robotics 思考过程可视化）。

## 4.8 小结

具身智能的核心技术堆栈正在快速汇聚到：

> **跨本体大规模 VLA / 世界模型 + 模仿学习预训练 + 强化学习微调 + GPU 并行仿真 + 安全约束控制层**

未来 2–3 年，竞争的焦点从"能不能做"变成"**谁的数据飞轮和工程化能力更强**"。
