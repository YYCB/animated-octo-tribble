# RealSense ROS 4.57.7 源码分析 - 节点工厂与设备管理

## 1. RealSenseNodeFactory 概述

`RealSenseNodeFactory` 是整个系统的入口点。它：
1. **是** ROS2 节点（继承 `RosNodeBase`，即 `rclcpp::Node` 或 `LifecycleNode`）
2. 负责设备发现、匹配和连接
3. 创建 `BaseRealSenseNode` 实例来处理实际数据流

### 文件位置
- 头文件：`include/realsense_node_factory.h`
- 实现：`src/realsense_node_factory.cpp`

## 4. 设备发现 (`getDevice()`)

`getDevice()` 方法遍历设备列表，按以下条件匹配：

```
对于每个设备 dev in list:
  1. 获取序列号 (RS2_CAMERA_INFO_SERIAL_NUMBER)
  2. 获取物理端口 (RS2_CAMERA_INFO_PHYSICAL_PORT) → parseUsbPort() 解析 USB 端口号
  3. 获取设备名称 (RS2_CAMERA_INFO_NAME)
  4. 获取产品 ID (RS2_CAMERA_INFO_PRODUCT_ID)

  匹配规则 (AND 关系):
  - serial_no 为空 OR serial_no == sn
  - usb_port_id 为空 OR usb_port_id == port_id
  - device_type 为空 OR 设备名正则匹配 device_type

  如果匹配成功:
    - 保存 _device = dev
    - 保存 _serial_no = sn
    - 检查 USB 类型 (USB 2.0 时发出性能警告)
```

## 5. 设备创建 (`startDevice()`)

```cpp
void RealSenseNodeFactory::startDevice() {
    // 1. 获取设备 PID
    uint16_t pid = std::stoi(pid_str, 0, 16);
    
    // 2. 根据 PID 创建 BaseRealSenseNode
    // 特殊处理: D555 的 PID 是 "DDS" 字符串
    switch(pid) {
        case RS400_PID: case RS405_PID: case RS410_PID: /*... 所有支持的 PID ...*/
            _realSenseNode = std::unique_ptr<BaseRealSenseNode>(
                new BaseRealSenseNode(*this, _device, _parameters, 
                    this->get_node_options().use_intra_process_comms()));
            break;
        default:
            ROS_FATAL_STREAM("Unsupported device!");
            exit(1);
    }
    
    // 3. 启动话题发布
    _realSenseNode->publishTopics();
}
```

**关键发现**：所有支持的设备类型都使用同一个 `BaseRealSenseNode` 类，没有按设备型号做子类区分。设备差异完全通过 librealsense2 SDK 内部处理。

## 6. 设备热插拔处理

```cpp
void RealSenseNodeFactory::changeDeviceCallback(rs2::event_information& info) {
    if (info.was_removed(_device)) {
        // 设备断开
        _realSenseNode.reset(nullptr);  // 销毁节点
        _device = rs2::device();        // 清空设备引用
    }
    if (!_device) {
        // 尝试获取新设备
        rs2::device_list new_devices = info.get_new_devices();
        if (new_devices.size() > 0) {
            getDevice(new_devices);
            if (_device) {
                startDevice();  // 重新创建节点
            }
        }
    }
}
```

## 7. Lifecycle Node 支持

当 `USE_LIFECYCLE_NODE` 编译选项开启时：

| 生命周期状态 | 回调方法 | 执行动作 |
|-------------|---------|---------|
| Unconfigured → Configuring | `on_configure()` | `init()` - 设备发现 |
| Inactive → Activating | `on_activate()` | `startDevice()` - 启动数据流 |
| Active → Deactivating | `on_deactivate()` | `stopDevice()` - 停止数据流 |
| Inactive → Cleaning Up | `on_cleanup()` | `closeDevice()` - 关闭设备 |
| Any → Shutting Down | `on_shutdown()` | `closeDevice()` - 关闭设备 |

```
                  init()          startDevice()
Unconfigured ──────────> Inactive ──────────> Active
                         <──────────         <──────────
                         closeDevice()       stopDevice()
```

**非 Lifecycle 模式**下，`init()` 中发现设备后直接调用 `startDevice()`，无需经过 Lifecycle 状态转换。

## 8. RSContextSingletonWrapper

```cpp

// 确保进程内只有一个 RS2 上下文
class RSContextSingletonWrapper {
```

使用单例模式确保：
1. 多个组件节点共享同一个 RS2 上下文
2. 设备变更回调只注册一次
3. 资源正确管理

## 9. 对 HAL 设计的启示

### 9.1 设备管理层接口建议

```
IDeviceManager:
  + discoverDevices() → vector<DeviceInfo>
  + connectDevice(filter: DeviceFilter) → IDevice
  + disconnectDevice(device: IDevice)
  + setDeviceChangeCallback(callback: function)
  + getContext() → SharedContext

DeviceFilter:
  + serialNumber: string (optional)
  + usbPort: string (optional)
  + deviceType: regex (optional)

DeviceInfo:
  + serialNumber: string
  + productId: uint16_t
  + deviceName: string
  + firmwareVersion: string
  + physicalPort: string
  + usbType: string
```

### 9.2 关键设计决策
1. **单一节点类**：realsense-ros 用一个 `BaseRealSenseNode` 处理所有设备类型，HAL 可以考虑是否需要按设备型号做多态
2. **热插拔**：需要支持设备断开重连，callback 机制是关键
3. **Lifecycle**：可选的生命周期管理对工业应用很重要
4. **Context 单例**：避免多实例冲突
