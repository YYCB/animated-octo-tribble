## 2. 类层次结构

### 2.1 节点类层次

```
rclcpp::Node (或 rclcpp_lifecycle::LifecycleNode)
  └── RosNodeBase  (类型别名，通过宏切换)
        └── RealSenseNodeFactory
              │
              └──owns──> BaseRealSenseNode (非 ROS Node，而是节点逻辑核心)
```

**关键设计**：`RosNodeBase` 是一个条件类型别名：
```cpp
#ifdef USE_LIFECYCLE_NODE
using RosNodeBase = rclcpp_lifecycle::LifecycleNode;
#else
using RosNodeBase = rclcpp::Node;
#endif
```

`RealSenseNodeFactory` **是** ROS 节点（继承 `RosNodeBase`），但 `BaseRealSenseNode` **不是** ROS 节点——它持有对 `RosNodeBase` 的引用，通过 `_node` 成员来操作 ROS API。

### 2.2 传感器类层次

```
rs2::sensor (librealsense2 SDK)
  └── RosSensor
        ├── SensorParams        (组合) - 管理传感器硬件参数
        ├── ProfilesManager[]   (组合) - 管理流配置
        │     ├── VideoProfilesManager  - 视频流 Profile
        │     └── MotionProfilesManager - 运动流 Profile
        └── FrequencyDiagnostics (组合) - 频率诊断
```

### 2.3 滤波器类层次

```
NamedFilter (基类)
  ├── PointcloudFilter    - 点云生成（继承 NamedFilter）
  └── AlignDepthFilter    - 深度对齐（继承 NamedFilter）
  
  (其他标准 rs2 滤波器通过 NamedFilter 直接包装)
```

### 2.4 参数类层次

```
Parameters
  └── ParametersBackend (ROS 参数服务器接口)

SensorParams
  └── 持有 shared_ptr<Parameters>
```

### 2.5 图像发布器层次

```
image_publisher (抽象基类)
  ├── image_rcl_publisher       - 原生 RCL 发布器（支持 intra-process）
  └── image_transport_publisher - image_transport 发布器（支持压缩传输）
```

## 4. 线程模型

```
主线程 (ROS2 Executor)
  ├── 服务回调处理
  ├── 参数更新回调
  └── Action 处理

查询线程 (_query_thread)
  └── 设备发现与连接（RealSenseNodeFactory::init 中创建）

Profile 监控线程 (_monitoring_pc)
  └── 等待 _cv_mpc 条件变量
  └── 调用 updateSensors() 处理 Profile 变更

温度监控线程 (_monitoring_t)
  └── 等待 _cv_temp 条件变量
  └── 发布温度诊断信息

动态 TF 线程 (_tf_t)
  └── 等待 _cv_tf 条件变量
  └── 按指定频率发布动态 TF

SDK 帧回调线程 (librealsense2 内部)
  └── 调用 frame_callback / imu_callback
  └── 通过 _asyncer 异步处理

参数更新线程 (_update_functions_t, Parameters 类内部)
  └── 等待 _update_functions_cv 条件变量
  └── 执行参数更新函数队列
```

## 5. 组件注册

节点通过 `rclcpp_components` 注册为可组合组件：

```cmake
rclcpp_components_register_node(realsense2_camera
  PLUGIN "realsense2_camera::RealSenseNodeFactory"
  EXECUTABLE realsense2_camera_node
)
```

这意味着：
1. 可以作为独立可执行程序运行
2. 可以通过组件容器加载（composition）
3. 支持 intra-process 通信（零拷贝）

## 6. 对 HAL 设计的架构启示

### 6.1 分层设计建议
```
HAL Layer 1: Device Discovery & Management  (对应 RealSenseNodeFactory)
HAL Layer 2: Sensor Abstraction             (对应 RosSensor)
HAL Layer 3: Stream Configuration           (对应 ProfilesManager)
HAL Layer 4: Data Pipeline                  (对应 frame_callback + Filters)
HAL Layer 5: Data Output                    (对应 Publishers，但不绑定 ROS)
```

### 6.2 关键抽象接口
- `IDeviceManager`: 设备发现、连接、断开、重连
- `ISensor`: 传感器打开、关闭、获取流配置
- `IStreamProfile`: 分辨率、帧率、格式
- `IFrameCallback`: 帧数据回调
- `IFilter`: 后处理滤波器
- `IParameterProvider`: 参数读取/写入（解耦 ROS）
