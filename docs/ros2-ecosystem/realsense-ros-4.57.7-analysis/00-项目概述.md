## 2. 项目定位

realsense-ros 是 Intel RealSense 深度相机的官方 ROS2 驱动包装器 (Wrapper)。它封装了 librealsense2 SDK，将 RealSense 设备的各种数据流（深度、彩色、红外、IMU、安全等）以标准 ROS2 话题/服务/Action 的形式对外发布。

## 4. 包 (Package) 组成

整个仓库由 **5 个 ROS2 包** 组成：

### 4.1 realsense2_camera (核心包)
- **功能**：主相机驱动节点，负责设备管理、数据流发布、参数管理、TF 变换等
- **类型**：C++ ROS2 组件节点 (Component Node)
- **构建系统**：ament_cmake

### 4.2 realsense2_camera_msgs (消息定义包)
- **功能**：定义自定义 ROS2 消息、服务和 Action
- **内容**：4 个 msg, 10 个 srv, 1 个 action
- **构建系统**：ament_cmake

### 4.3 realsense2_description (URDF 描述包)
- **功能**：提供 RealSense 相机的 URDF 模型和 3D 网格文件
- **内容**：URDF/Xacro 文件、STL 网格、Launch 和 RViz 配置
- **构建系统**：ament_cmake

### 4.4 realsense2_rgbd_plugin (RGBD 插件包)
- **功能**：为 RGBD 相机提供 Nodelet/Plugin 支持
- **状态**：包含 `CATKIN_IGNORE` 文件（默认不构建）
- **构建系统**：ament_cmake

### 4.5 realsense2_ros_mqtt_bridge (MQTT 桥接包)
- **功能**：ROS2 到 MQTT 的桥接，支持远程数据传输
- **状态**：包含 `CATKIN_IGNORE` 文件（默认不构建）
- **语言**：Python
- **构建系统**：ament_python

## 5. 核心依赖

### 5.1 SDK 依赖
- **librealsense2** >= 2.57.7（必需）
- **librealsense2-gl** >= 2.57.7（可选，GPU 加速）

### 5.2 ROS2 依赖
- rclcpp / rclcpp_components / rclcpp_action
- sensor_msgs / std_msgs / std_srvs / nav_msgs
- tf2 / tf2_ros
- cv_bridge / image_transport
- diagnostic_updater
- rclcpp_lifecycle / lifecycle_msgs（可选，Lifecycle Node 模式）

### 5.3 第三方依赖
- OpenCV (core)
- Eigen3 (Geometry)

## 7. 分析目标

本次源码分析的最终目标是：**基于 realsense-ros 的实现，设计一个硬件抽象层 (HAL)**。因此需要重点关注：

1. **设备抽象方式**：SDK 如何抽象不同型号的相机
2. **传感器管理**：多传感器（深度、彩色、红外、IMU、安全）的统一管理模式
3. **数据流架构**：从设备到 ROS 话题的完整数据管道
4. **参数系统**：动态参数如何映射 SDK 选项
5. **Profile 管理**：流配置/分辨率/帧率的管理方式
6. **滤波器管道**：后处理滤波器的可插拔架构
7. **坐标变换**：多传感器间的外参管理和 TF 发布
8. **生命周期管理**：设备连接/断开/重连的处理机制
