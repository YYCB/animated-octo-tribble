# RealSense ROS 4.57.7 源码分析 - 参数系统详解

## 1. 概述

参数系统分为三层：
1. **Parameters** (顶层)：统一的参数管理器，管理所有 ROS 参数
2. **SensorParams** (传感器层)：管理传感器硬件选项（exposure, gain 等）
3. **ParametersBackend** (底层)：ROS2 参数服务器的封装

### 文件位置
- `include/dynamic_params.h` / `src/dynamic_params.cpp` - Parameters 类
- `include/sensor_params.h` / `src/sensor_params.cpp` - SensorParams 类
- `include/ros_param_backend.h` / `src/ros_param_backend.cpp` - ParametersBackend 类

## 2. Parameters 类

### 2.1 核心成员

```cpp
class Parameters {
private:
    RosNodeBase& _node;                        // ROS 节点引用
    rclcpp::Logger _logger;
    
    // 参数名 → 回调函数映射
    map<string, function<void(const rclcpp::Parameter&)>> _param_functions;
    
    // 参数值指针 → 参数名映射（用于双向查找）
    map<void*, string> _param_names;
    
    // ROS 参数后端
    ParametersBackend _params_backend;
    
    // 参数更新队列
    condition_variable _update_functions_cv;
    bool _is_running;
    shared_ptr<thread> _update_functions_t;      // 更新线程
    deque<function<void()>> _update_functions_v;  // 更新函数队列
    list<string> self_set_parameters;             // 自设参数列表
    mutex _mu;
};
```

### 2.2 参数声明与设置

```cpp
// 声明并设置一个 ROS 参数
template <class T>
T setParam(string param_name, const T& initial_value,
           function<void(const rclcpp::Parameter&)> func = nullptr,
           ParameterDescriptor descriptor = {});
```

流程：
1. 如果参数已存在（已被 launch 文件设置），读取现有值
2. 如果不存在，用 `initial_value` 声明参数
3. 如果提供了 `func` 回调，注册到 `_param_functions`
4. 返回参数的当前值

```cpp
// 将参数绑定到变量
template <class T>
void setParamT(string param_name, T& param,
               function<void(const rclcpp::Parameter&)> func = nullptr,
               ParameterDescriptor descriptor = {});
```

与 `setParam` 类似，但直接修改传入的引用变量。

### 2.3 参数更新机制

```
用户通过 ros2 param set 修改参数
        │
        ▼
ParametersBackend::on_set_parameters_callback
        │
        ├── 检查参数是否是自设的 (self_set_parameters)
        │     └── 是 → 跳过回调（避免循环）
        │
        └── 查找 _param_functions[param_name]
              │
              └── 调用注册的回调函数
                    例如: update_sensor_func → 触发 Profile 重新配置
```

### 2.4 异步参数更新

```cpp
void pushUpdateFunctions(vector<function<void()>> funcs) {
    lock_guard lock(_mu);
    for (auto& f : funcs)
        _update_functions_v.push_back(f);
    _update_functions_cv.notify_one();
}
```

有一个专门的线程 `_update_functions_t` 消费更新函数队列：
```cpp
void monitor_update_functions() {
    while (_is_running) {
        unique_lock lock(_mu);
        _update_functions_cv.wait(lock, [&]{ return !_is_running || !_update_functions_v.empty(); });
        
        while (!_update_functions_v.empty()) {
            auto func = _update_functions_v.front();
            _update_functions_v.pop_front();
            lock.unlock();
            func();  // 执行更新
            lock.lock();
        }
    }
}
```

### 2.5 参数值反向设置

## 3. SensorParams 类

### 3.1 职责

将 librealsense2 传感器的硬件选项（`rs2_option`）自动映射为 ROS 参数。

### 3.2 选项类型判断

```cpp
// 判断选项类型
bool is_checkbox(sensor, option);   // range=[0,1], step=1 → bool
bool is_enum_option(sensor, option); // 有字符串描述 → enum (string)
bool is_int_option(sensor, option);  // step=1 → int
// 否则 → double (float)
```

### 3.3 动态选项注册

```cpp
void SensorParams::registerDynamicOptions(rs2::options sensor, const string& module_name) {
    auto supported_options = sensor.get_supported_options();
    
    for (auto option : supported_options) {
        if (sensor.is_option_read_only(option))
            continue;  // 跳过只读选项
        
        string option_name = module_name + "." + ros_option_name(option);
        
        // 根据类型注册
        if (is_checkbox(sensor, option)) {
            set_parameter<bool>(sensor, option, module_name);
        }
        else if (is_enum_option(sensor, option)) {
            // 枚举类型用 int 表示，描述中列出可选值
            set_parameter<int>(sensor, option, module_name, descriptions);
        }
        else if (is_int_option(sensor, option)) {
            set_parameter<int>(sensor, option, module_name);
        }
        else {
            set_parameter<double>(sensor, option, module_name);
        }
    }
}
```

### 3.4 参数描述生成

```cpp
template<class T>
ParameterDescriptor get_parameter_descriptor(
    const string& option_name,
    rs2::option_range option_range,
    T option_value,
    const string& option_description,
    const string& description_addition)
{
    ParameterDescriptor desc;
    desc.name = option_name;
    
    if (T == double) {
        // FloatingPointRange
        desc.floating_point_range = [{min, max, step}];
    } else if (T == bool) {
        // IntegerRange [0, 1]
        desc.integer_range = [{0, 1, 1}];
    } else {
        // IntegerRange
        desc.integer_range = [{min, max, step}];
    }
    
    desc.description = option_description + "\nDefault value: " + default;
    return desc;
}
```

### 3.5 参数回调

当用户修改参数时：
```cpp
// 在 set_parameter 中注册的回调
[sensor, option](const rclcpp::Parameter& parameter) {
    sensor.set_option(option, parameter.get_value<T>());
}
```

即直接将 ROS 参数值写入 SDK 传感器选项。

### 3.6 精度处理

```cpp
double SensorParams::float_to_double(float val, option_range) {
    // 从 float 转 double 时精度对齐
    // 确保值严格落在 [min, max] 范围内的步进点上
    float range = option_range.max - option_range.min;
    int steps = range / option_range.step;
    double step = double(range) / steps;
    double rounded_div = round((val - min) / step);
    return min + rounded_div * step;
}
```

这是因为 ROS2 参数验证对 double 值的精度非常敏感。

## 4. ParametersBackend 类

### 4.1 参数变更回调

```cpp
// 注册全局参数变更回调
rcl_interfaces::msg::SetParametersResult on_set_parameters_callback(
    const vector<rclcpp::Parameter>& params)
{
    for (auto& parameter : params) {
        auto name = parameter.get_name();
        
        // 检查是否是自己设置的（避免循环回调）
        if (self_set_parameters.contains(name)) {
            self_set_parameters.remove(name);
            continue;
        }
        
        // 调用注册的回调
        if (_param_functions.contains(name)) {
            _param_functions[name](parameter);
        }
    }
    return result;  // success
}
```

## 7. 对 HAL 设计的启示

### 7.1 参数层抽象

```
IParameterProvider:
  + getParameter<T>(name, default) → T
  + setParameter<T>(name, value)
  + registerCallback(name, callback)
  + removeParameter(name)
  + listParameters() → vector<ParameterInfo>

ParameterInfo:
  + name: string
  + type: ParameterType (bool/int/double/string)
  + range: Range (min, max, step)
  + description: string
  + currentValue: any
  + defaultValue: any

// ROS 实现
class RosParameterProvider : IParameterProvider { ... }
// 配置文件实现
class FileParameterProvider : IParameterProvider { ... }
```

### 7.2 关键设计决策
1. **自动注册**：SDK 选项自动映射为参数，HAL 应该保留这个设计
2. **双向同步**：参数变更 → SDK，SDK 变更 → 参数（如 HDR 禁用自动曝光）
3. **异步更新**：避免在回调中直接操作传感器，通过队列异步执行
4. **精度处理**：float↔double 转换需要小心处理
5. **参数解耦**：HAL 的参数层不应绑定 ROS2
