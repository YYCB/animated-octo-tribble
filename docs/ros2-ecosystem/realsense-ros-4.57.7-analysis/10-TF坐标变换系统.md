# RealSense ROS 4.57.7 源码分析 - TF 坐标变换系统

## 1. 概述

TF（Transform）系统管理 RealSense 设备内各传感器之间的空间关系，并将其以 ROS TF2 标准发布。

### 文件位置
- `src/tfs.cpp` (13KB) - TF 变换发布

## 2. 坐标系定义

### 2.1 坐标系命名规则

```
{tf_prefix}{camera_name}_{stream_name}_frame          # ROS 坐标系
{tf_prefix}{camera_name}_{stream_name}_optical_frame   # 光学坐标系
{tf_prefix}{base_frame_id}                             # 基准坐标系
{tf_prefix}{camera_name}_imu_frame                     # IMU 坐标系
{tf_prefix}{camera_name}_imu_optical_frame             # IMU 光学坐标系
```

### 2.2 宏定义

```cpp
#define BASE_FRAME_ID       "{tf_prefix}{base_frame_id}"
#define FRAME_ID(sip)       "{tf_prefix}{camera_name}_{stream_name}_frame"
```

## 3. 坐标系约定

### 3.1 ROS 坐标系 (ROS CS)
```
X: 前方 (Forward)
Y: 左方 (Left)
Z: 上方 (Up)
```

### 3.2 光学坐标系 (Optical CS)
```
X: 右方 (Right)
Y: 下方 (Down)
Z: 前方 (Forward, 朝相机光轴方向)
```

### 3.3 ROS → Optical 变换

旋转四元数：
```cpp
tf2::Quaternion quaternion_optical;
quaternion_optical.setRPY(-M_PI/2, 0, -M_PI/2);
// Roll=-90°, Pitch=0°, Yaw=-90°
```

这个旋转将 ROS CS 变换为 Optical CS。

## 4. 外参计算与 TF 生成 (`calcAndAppendTransformMsgs()`)

### 4.1 基本流程

```
对于每个传感器流 Profile:
  │
  ├── 1. 获取外参
  │     ├── normal_ex = base_profile → profile  (用于外参话题)
  │     └── tf_ex = profile → base_profile      (用于 TF)
  │
  ├── 2. 将旋转矩阵转为四元数
  │     └── Q = rotationMatrixToQuaternion(tf_ex.rotation)
  │
  ├── 3. 根据流类型计算最终变换
  │     │
  │     ├── [Labeled Point Cloud / Occupancy]
  │     │     Q_final = Q * quaternion_optical
  │     │     (光学CS直接到机器人CS)
  │     │
  │     └── [其他流]
  │           Q_final = quaternion_optical * Q * quaternion_optical.inverse()
  │           (ROS→光学→外参→光学→ROS)
  │
  ├── 4. 添加 TF: BASE_FRAME → {stream}_frame
  │     └── 平移: 光学CS → ROS CS 转换
  │           x_ros = z_optical
  │           y_ros = -x_optical
  │           z_ros = -y_optical
  │
  ├── 5. 添加 TF: {stream}_frame → {stream}_optical_frame
  │     └── 纯旋转 (quaternion_optical), 零平移
  │
  ├── 6. [对齐深度额外 TF]
  │     ├── BASE_FRAME → aligned_depth_to_{stream}_frame
  │     └── aligned_depth_to_{stream}_frame → {stream}_optical_frame
  │
  └── 7. [IMU 同步额外 TF]
        ├── gyro_frame → imu_frame (零旋转)
        └── imu_frame → imu_optical_frame (quaternion_optical)
```

### 4.2 旋转矩阵到四元数

```cpp
tf2::Quaternion BaseRealSenseNode::rotationMatrixToQuaternion(const float rotation[9]) const {
    Eigen::Matrix3f m;
    // RS2 旋转矩阵是列主序
    m << rotation[0], rotation[3], rotation[6],
         rotation[1], rotation[4], rotation[7],
         rotation[2], rotation[5], rotation[8];
    Eigen::Quaternionf q(m);
    return tf2::Quaternion(q.x(), q.y(), q.z(), q.w());
}
```

### 4.3 平移向量坐标转换

在 `append_static_tf_msg` 中：
```cpp
// 从光学坐标系转换到 ROS 坐标系
msg.transform.translation.x = trans.z;    // 光学Z → ROS X
msg.transform.translation.y = -trans.x;   // 光学-X → ROS Y
msg.transform.translation.z = -trans.y;   // 光学-Y → ROS Z
```

## 5. TF 发布方式

### 5.1 静态 TF (`publishStaticTransforms()`)

```cpp
void BaseRealSenseNode::publishStaticTransforms() {
    restartStaticTransformBroadcaster();  // 重建 broadcaster
    _static_tf_broadcaster->sendTransform(_static_tf_msgs);
}
```

**注意**：每次传感器变更时重建 `StaticTransformBroadcaster`，因为 latched 话题的旧消息会保留。

```cpp
void BaseRealSenseNode::restartStaticTransformBroadcaster() {
    _static_tf_broadcaster.reset();  // 先销毁旧的
    
    // 禁用 intra-process（latched QoS 不支持 intra-process）
    PublisherOptions options;
    options.use_intra_process_comm = IntraProcessSetting::Disable;
    
    _static_tf_broadcaster = make_shared<StaticTransformBroadcaster>(
        _node, StaticBroadcasterQoS(), std::move(options));
}
```

### 5.2 动态 TF (`publishDynamicTransforms()`)

当 `tf_publish_rate > 0` 时启用：

```cpp
void BaseRealSenseNode::publishDynamicTransforms() {
    _dynamic_tf_broadcaster = make_shared<TransformBroadcaster>(_node);
    
    while (rclcpp::ok() && _is_running && _tf_publish_rate > 0) {
        // 等待指定频率
        _cv_tf.wait_for(lock, chrono::milliseconds(1000.0/_tf_publish_rate));
        
        // 更新时间戳并发布
        rclcpp::Time t = _node.now();
        for (auto& msg : _static_tf_msgs)
            msg.header.stamp = t;
        _dynamic_tf_broadcaster->sendTransform(_static_tf_msgs);
    }
}
```

**用途**：某些下游节点需要在 `/tf`（非 `/tf_static`）上接收变换。

## 6. TF 的动态管理

### 6.1 添加 TF

```cpp
void append_static_tf_msg(time, trans, quaternion, from_frame, to_frame);
```

### 6.2 删除 TF（传感器禁用时）

```cpp
void erase_static_tf_msg(frame_id, child_frame_id) {
    // 查找并删除匹配的 TF 消息
    auto it = find_if(_static_tf_msgs.begin(), _static_tf_msgs.end(),
                      compare_tf_ids(frame_id, child_frame_id));
    if (it != _static_tf_msgs.end())
        _static_tf_msgs.erase(it);
}

void eraseTransformMsgs(sip, profile) {
    erase_static_tf_msg(BASE_FRAME_ID, FRAME_ID(sip));
    erase_static_tf_msg(FRAME_ID(sip), OPTICAL_FRAME_ID(sip));
    // ... 以及对齐和 IMU 的额外 TF
}
```

## 7. 外参话题发布

除了 TF，还会发布自定义外参消息：

```cpp
void publishExtrinsicsTopic(const stream_index_pair& sip, const rs2_extrinsics& ex) {
    Extrinsics msg = rsExtrinsicsToMsg(ex);  // 9个旋转 + 3个平移
    _extrinsics_publishers[sip]->publish(msg);
}

// 话题名: ~/extrinsics/{base_stream}_to_{stream}
// 例如: ~/extrinsics/depth_to_color
```

外参消息格式：
```
Extrinsics.msg:
  float64[9] rotation     # 列主序 3x3 旋转矩阵
  float64[3] translation  # 平移向量（米）
```

## 9. 对 HAL 设计的启示

### 9.1 坐标系管理接口

```
IExtrinsicsProvider:
  + getExtrinsics(from: StreamProfile, to: StreamProfile) → Extrinsics
  + getIntrinsics(profile: StreamProfile) → Intrinsics

ITransformManager:
  + addTransform(parent, child, transform: Transform)
  + removeTransform(parent, child)
  + getTransformTree() → vector<Transform>
  + setBaseFrame(frame_id: string)

Transform:
  + parentFrame: string
  + childFrame: string
  + rotation: Quaternion
  + translation: Vector3
  + timestamp: Time

Intrinsics:
  + width, height: int
  + fx, fy, ppx, ppy: double
  + distortionModel: DistortionModel
  + distortionCoeffs: double[5]
```

### 9.2 关键设计决策
1. **双坐标系**：每个传感器同时有 ROS 和 Optical 两个坐标系
2. **基准流**：所有 TF 以 base_frame 为根，通常是深度传感器
3. **动态管理**：传感器启停会添加/删除 TF
4. **latched 发布**：静态 TF 使用 transient_local 持久化
5. **外参双用**：既用于 TF 也用于独立的外参话题
