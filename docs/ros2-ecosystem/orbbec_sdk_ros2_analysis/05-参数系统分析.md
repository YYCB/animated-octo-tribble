# 参数系统深度分析

## 1. 参数架构

### 1.1 三层参数体系

```
Launch 文件参数（DeclareLaunchArgument）← 优先级最高，命令行可覆盖
    │
    ▼
YAML 配置文件参数（config_file_path 指定）
    │
    ▼
代码默认值（constants.h / 代码硬编码）← 优先级最低
    │
    ▼
最终参数值
```

### 1.2 参数加载流程（Launch 文件）

```python
def load_parameters(context, args):
    # 1. 收集所有 Launch 参数默认值
    default_params = {arg.name: LaunchConfiguration(arg.name)}

    # 2. 如果指定了 config_file_path，加载 YAML 覆盖
    if config_file_path:
        yaml_params = load_yaml(config_file_path)
        default_params = merge_params(default_params, yaml_params)

    # 3. 类型转换（字符串 → int/float/bool）
    return converted_params
```

## 3. 动态参数变更机制

```cpp
// Parameters::setParam() 注册变更回调
parameters_->setParam("color_exposure",
    rclcpp::ParameterValue(color_exposure_),
    [this](const rclcpp::Parameter& param) {
        color_exposure_ = param.as_int();
        device_->setIntProperty(OB_PROP_COLOR_EXPOSURE_INT, color_exposure_);
    });
```

运行时可通过 `ros2 param set` 命令修改参数，回调自动应用到硬件。

## 4. 参数对 HAL 设计的影响

### 4.1 需要在 HAL 层抽象的核心参数
- 流使能（enable_color / enable_depth / enable_ir）
- 分辨率与帧率（width / height / fps）
- 像素格式（format）
- 曝光 / 增益控制
- 深度对齐（depth_registration / align_mode）
- 时间戳域（time_domain）

### 4.2 设备特定参数
| 参数 | 仅适用设备 |
|------|----------|
| depth_work_mode | Gemini 2 系列 |
| enable_gmsl_trigger | GMSL 接口设备 |
| depth_filter_config | Gemini 2 系列 |
