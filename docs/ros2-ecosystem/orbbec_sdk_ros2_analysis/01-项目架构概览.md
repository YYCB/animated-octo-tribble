# OrbbecSDK_ROS2 v2.7.6 项目架构概览

## 1. 项目概述

- Orbbec（奥比中光）为其深度相机和 LiDAR 产品线提供的 ROS2 封装驱动
- 支持 ROS2 Foxy / Humble / Jazzy
- v2 版本基于 OrbbecSDK v2（libobsensor），区别于 v1 的 OpenNI 后端
- 许可证：Apache 2.0

## 2. 仓库顶层结构

```
OrbbecSDK_ROS2/
├── orbbec_camera/               # 主驱动包（核心代码）
│   ├── CMakeLists.txt

│   ├── launch/                  # 各设备系列的 Launch 文件
│   ├── config/                  # YAML 配置文件

│   ├── scripts/                 # 脚本（udev 规则等）
│   └── rviz/                    # RViz 配置

```

## 4. 关键设计模式

### 4.1 组件化架构（Composable Node）
`OBCameraNodeDriver` 使用 `rclcpp_components` 注册为组件节点，支持：
- 独立进程运行（`ros2 run`）
- 组件容器运行（减少进程间通信开销）
- Intra-process 通信（零拷贝图像传输）

### 4.2 双节点设计
根据设备类型动态选择：
- **OBCameraNode**：处理深度相机类设备（Gemini、Femto、Astra 系列）
- **OBLidarNode**：处理 LiDAR 类设备（Pulsar ME450/SL450）

### 4.3 设备发现策略（四种方式）
1. **序列号匹配**（`serial_number` 参数）
2. **USB 端口匹配**（`usb_port` 参数）
3. **网络 IP 匹配**（`net_device_ip` 参数）
4. **自动选择**（默认选择第一个可用设备）

### 4.4 Pipeline 数据流模型
- **主 Pipeline**：处理所有图像流（color、depth、IR）
- **IMU Pipeline**：独立 Pipeline 处理 IMU 数据

### 4.5 多设备支持
- 共享内存进程锁（`orb_device_lock`）
- 多相机 Launch 配置
- 硬件同步支持（`OBMultiDeviceSyncMode`）

## 5. 构建系统（CMakeLists.txt）

### 5.1 编译标志
- C++17 标准
- 严格警告：`-Wall -Wextra -Werror`
- 优化：`-O3`

### 5.2 平台支持
- x86_64（x64）
- aarch64（arm64）
- 不支持 arm32

### 5.3 可选硬件解码器
- `USE_RK_HW_DECODER`：Rockchip MPP 硬件解码（RK3588 等）
- `USE_NV_HW_DECODER`：NVIDIA Jetson 硬件解码

### 5.4 主要依赖
- ROS2：rclcpp、sensor_msgs、image_transport、cv_bridge、tf2、diagnostic_updater
- 第三方：OpenCV、Eigen3、nlohmann-json、OpenSSL、yaml-cpp、magic_enum
