# 13 — 构建与平台适配对比

> **本章回答**:两家的 CMake 结构长什么样?依赖哪些外部库?有哪些编译选项?如何支持 Jetson/RK 等嵌入式平台?HAL 应该怎么组织构建?

---

## 1. Orbbec 侧:多 CMake 选项 + 三平台硬件解码

### 1.1 Package 结构

```
OrbbecSDK_ROS2/
  orbbec_camera/                # 主驱动
    CMakeLists.txt
    include/
    src/
      ob_camera_node_driver.cpp
      ob_camera_node.cpp        # 主业务逻辑(~220KB)
      ob_lidar_node.cpp
      jpeg_decoder.cpp
      jetson_nv_decoder.cpp     # 条件编译
      rk_mpp_decoder.cpp         # 条件编译
      parameters.cpp
      image_publisher.cpp
      ...
    launch/
    config/
    cmake/
  orbbec_camera_msgs/           # 自定义消息
  orbbec_camera_srv/            # 自定义服务
```

### 1.2 外部依赖

| 库 | 用途 |
|---|---|
| **OrbbecSDK v2 (libobsensor)** | 核心 SDK,自带或系统安装 |
| rclcpp / rclcpp_components | ROS2 节点 |
| sensor_msgs / geometry_msgs / tf2_ros | 消息与 TF |
| image_transport | 图像发布 |
| cv_bridge / OpenCV | 图像处理 |
| Eigen3 | 线性代数 |
| nlohmann_json | JSON 滤波器状态 |
| libjpeg-turbo | MJPEG 软件解码 |
| libuvc(间接) | USB 后端 |

### 1.3 关键 CMake 选项

```cmake
option(USE_RK_HW_DECODER     "Enable Rockchip hardware decoder" OFF)
option(USE_NV_HW_DECODER     "Enable NVIDIA hardware decoder"   OFF)
option(USE_GMSL              "Enable GMSL support on Jetson"     OFF)
option(ORB_USE_SDK_FROM_SOURCE "Build SDK from source"            OFF)
option(BUILD_TESTING         "Build tests"                        OFF)
```

### 1.4 Jetson 硬件解码

```cmake
if(USE_NV_HW_DECODER)
    find_package(CUDA REQUIRED)
    # Jetson Multimedia API:nvbuf_utils, nvjpeg, nvosd
    target_link_libraries(${PROJECT_NAME}
        ${CUDA_LIBRARIES}
        nvbufsurface
        nvjpeg
        ...)
    target_compile_definitions(${PROJECT_NAME} PRIVATE USE_NV_HW_DECODER)
endif()
```

源代码中:
```cpp
#ifdef USE_NV_HW_DECODER
    decoder_ = std::make_unique<JetsonNVDecoder>();
#elif defined(USE_RK_HW_DECODER)
    decoder_ = std::make_unique<RkMppDecoder>();
#else
    decoder_ = std::make_unique<JPEGDecoder>();
#endif
```

**后果**:二进制绑定到平台,一次编译只能跑在一个平台上。

### 1.5 RK 硬件解码

```cmake
if(USE_RK_HW_DECODER)
    find_path(RKRGA_INCLUDE_DIR rga.h)
    find_library(RKRGA_LIBRARY rga)
    find_library(RKMPP_LIBRARY rockchip_mpp)
    # ...
endif()
```

### 1.6 GMSL(Jetson + 特定硬件)

```cmake
if(USE_GMSL)
    # 加载 gmsl_loader.py 脚本处理 Jetson v4l2 配置
    # 编译路径含 /opt/nvidia/jetson-multimedia-api
endif()
```

### 1.7 SDK 预编译 vs 源码编译

```cmake
if(ORB_USE_SDK_FROM_SOURCE)
    add_subdirectory(OrbbecSDK)
else()
    # 使用预编译 so (linux-x86_64 / linux-aarch64 / ...)
    find_library(ORBBEC_SDK NAMES OrbbecSDK HINTS ...)
endif()
```

Orbbec 在仓库中 **vendor 了预编译 SDK** 二进制到各个平台目录下:
```
OrbbecSDK/
  linux_x64/
    libOrbbecSDK.so.2.7.6
  linux_aarch64/
    libOrbbecSDK.so.2.7.6
  windows_x64/
    OrbbecSDK.dll
```

### 1.8 Udev 规则

安装脚本 `99-obsensor-libusb.rules` 复制到 `/etc/udev/rules.d/` 允许非 root 访问设备。

## 2. RealSense 侧:单 CMake + 最小平台适配

### 2.1 Package 结构

```
realsense-ros/
  realsense2_camera/
    CMakeLists.txt
    include/
    src/
      realsense_node_factory.cpp
      base_realsense_node.cpp
      rs_node_setup.cpp
      tfs.cpp
      safety.cpp
      actions.cpp
      parameters.cpp
      named_filter.cpp
      profile_manager.cpp
      ...
    launch/
    config/
  realsense2_camera_msgs/
  realsense2_description/       # URDF
```

### 2.2 外部依赖

| 库 | 用途 |
|---|---|
| **librealsense2** (>= 2.57) | 核心 SDK,系统安装 |
| rclcpp / rclcpp_lifecycle | ROS2 节点(含 Lifecycle) |
| rclcpp_components | Composable |
| sensor_msgs / ... | 消息与 TF |
| image_transport | 图像发布(可选) |
| diagnostic_updater | 诊断 |
| tf2_ros | TF 发布 |

### 2.3 关键 CMake 选项

```cmake
option(BUILD_ACCELERATE_GPU_WITH_GLSL "Enable GLSL acceleration" OFF)
option(BUILD_TOOLS  "Build extra tools" OFF)
option(USE_LIFECYCLE_NODE "Use LifecycleNode" OFF)
```

**比 Orbbec 少得多**:因为 librealsense2 自己处理了平台差异(Linux/Windows/Mac/Jetson),Wrapper 无需分化。

### 2.4 Lifecycle 编译切换

```cmake
if(USE_LIFECYCLE_NODE)
    target_compile_definitions(${PROJECT_NAME} PRIVATE USE_LIFECYCLE_NODE)
    target_link_libraries(${PROJECT_NAME} rclcpp_lifecycle::rclcpp_lifecycle)
endif()
```

源代码:
```cpp
#ifdef USE_LIFECYCLE_NODE
using RosNodeBase = rclcpp_lifecycle::LifecycleNode;
#else
using RosNodeBase = rclcpp::Node;
#endif
```

### 2.5 GLSL 加速

```cmake
if(BUILD_ACCELERATE_GPU_WITH_GLSL)
    find_package(OpenGL REQUIRED)
    find_package(GLEW  REQUIRED)
    target_link_libraries(${PROJECT_NAME} OpenGL::GL GLEW::GLEW)
endif()
```

Wrapper 层只是开关 SDK 能力,加速由 SDK GLSL shader 实现。

### 2.6 SDK 安装方式

通过 Intel apt repo 或源码编译 librealsense2 到系统,ROS wrapper 通过 `find_package(realsense2 REQUIRED)` 引入。

### 2.7 Udev 规则

通过 librealsense2 安装包自动部署。

## 3. 对比总览

| 维度 | Orbbec | RealSense |
|---|---|---|
| CMake 选项数 | ~5(含硬件解码) | ~3 |
| 硬件解码器切换 | 编译期(三选一) | ❌ 不需要 |
| Lifecycle 切换 | ❌ | 编译期 |
| SDK 分发方式 | **仓库内 vendor** | 系统 apt |
| SDK 构建 | 可选从源码 | 不需要(用预安装) |
| Jetson 支持 | **专门优化**(NV decoder + GMSL) | 通过 librealsense2 透明支持 |
| RK 平台 | **专门优化**(RK MPP) | 需自编译 librealsense2 |
| Udev | 手工脚本 | SDK 包自带 |
| 单文件大小 | 最大 220 KB(主业务) | 最大 55 KB |
| 编译时间 | 长(含 JPEG + HW 解码) | 短 |

### ⚠️ 关键差异

1. **平台适配策略**:
   - RealSense:让 SDK 吃掉所有平台差异,Wrapper 平台无关。
   - Orbbec:SDK 本身平台无关,但硬件解码器需要 Wrapper 侧分化。
   - **HAL 决策**:HAL 接口本身平台无关,**解码器插件** 在各自 CMake 选项下编译。

2. **SDK 分发**:
   - Orbbec vendor 预编译 SO 提高"开箱即用",但更新 SDK 要修改仓库。
   - RealSense 依赖系统安装,SDK 与 Wrapper 独立升级。
   - **HAL 决策**:后端 Backend 各自 find_package,不 vendor。允许用户装不同版本。

## 4. 对 HAL 的可复用要点

### 4.1 建议的 Package 结构

```
camera_hal/
  core/                              # 接口定义(仅头文件 or 静态库)
    include/camera_hal/
      i_device_manager.h
      i_device.h
      i_sensor.h
      i_frame.h
      i_filter.h
      ...
    src/  (空或 basic 工具类)
    CMakeLists.txt → libcamera_hal_core.so (或纯 interface library)

  backend_orbbec/                    # Orbbec 后端
    include/
    src/
      orbbec_device_manager.cpp
      orbbec_device.cpp
      orbbec_sensor.cpp
      orbbec_filter.cpp
    CMakeLists.txt → libcamera_hal_orbbec.so
    plugins.xml

  backend_realsense/                 # RealSense 后端
    src/
      realsense_device_manager.cpp
      realsense_device.cpp
      realsense_sensor.cpp
    CMakeLists.txt → libcamera_hal_realsense.so
    plugins.xml

  decoders/                          # 解码器插件
    libjpeg_turbo/
    jetson_nv/
    rk_mpp/
    CMakeLists.txt(每个 decoder 条件编译)

  ros_adapter/                       # ROS Adapter 层
    src/
      hal_camera_driver.cpp          # ROS 节点(相当于 OBCameraNodeDriver)
      hal_camera_logic.cpp           # 业务逻辑(相当于 OBCameraNode)
      ros_timestamp_converter.cpp
      ros_tf_publisher.cpp
    include/
    launch/
    config/

  msgs/
  tests/
    unit_tests/
    integration_tests/
```

### 4.2 CMake 分层

```cmake
# camera_hal/core/CMakeLists.txt
add_library(camera_hal_core INTERFACE)
target_include_directories(camera_hal_core INTERFACE include)
# 纯接口,无实现依赖

# camera_hal/backend_orbbec/CMakeLists.txt
option(BUILD_ORBBEC_BACKEND "Build Orbbec HAL backend" ON)
if(BUILD_ORBBEC_BACKEND)
    find_package(OrbbecSDK REQUIRED)
    add_library(camera_hal_orbbec SHARED ...)
    target_link_libraries(camera_hal_orbbec camera_hal_core OrbbecSDK::OrbbecSDK)
endif()

# camera_hal/backend_realsense/CMakeLists.txt
option(BUILD_REALSENSE_BACKEND "Build RealSense HAL backend" ON)
if(BUILD_REALSENSE_BACKEND)
    find_package(realsense2 REQUIRED)
    add_library(camera_hal_realsense SHARED ...)
    target_link_libraries(camera_hal_realsense camera_hal_core realsense2::realsense2)
endif()

# camera_hal/decoders/CMakeLists.txt
option(BUILD_DECODER_JPEG_TURBO "Build libjpeg-turbo decoder" ON)
option(BUILD_DECODER_JETSON_NV  "Build Jetson NV decoder"     OFF)
option(BUILD_DECODER_RK_MPP     "Build RK MPP decoder"        OFF)

if(BUILD_DECODER_JPEG_TURBO)
    find_package(JPEG REQUIRED)
    add_library(camera_hal_decoder_jpeg_turbo SHARED libjpeg_turbo/*.cpp)
endif()

if(BUILD_DECODER_JETSON_NV)
    find_package(CUDA REQUIRED)
    find_library(NVJPEG nvjpeg)
    find_library(NVBUFSURFACE nvbufsurface)
    add_library(camera_hal_decoder_jetson SHARED jetson_nv/*.cpp)
endif()

# 自动检测平台
if(NOT DEFINED BUILD_DECODER_JETSON_NV)
    if(EXISTS "/etc/nv_tegra_release")
        set(BUILD_DECODER_JETSON_NV ON CACHE BOOL "Auto-detected Jetson")
    endif()
endif()
```

### 4.3 后端加载方式(运行时)

两选:

**方式 A:静态链接**(简单,但需要所有后端同时可用才行)
```cmake
target_link_libraries(camera_hal_ros_adapter
    camera_hal_orbbec camera_hal_realsense)
```
工厂注册:
```cpp
// 静态初始化每个后端
static BackendRegistrar orbbec_reg("orbbec", &createOrbbecManager);
static BackendRegistrar realsense_reg("realsense", &createRealSenseManager);
```

**方式 B:pluginlib / dlopen 动态加载**(灵活,运行时选后端)
```cmake
pluginlib_export_plugin_description_file(camera_hal plugins.xml)
```
plugins.xml:
```xml
<library path="camera_hal_orbbec">
  <class type="camera_hal::OrbbecDeviceManager"
         base_class_type="camera_hal::IDeviceManager"
         name="orbbec">
  </class>
</library>
```
工厂:
```cpp
auto loader = pluginlib::ClassLoader<IDeviceManager>("camera_hal", "camera_hal::IDeviceManager");
auto dm = loader.createSharedInstance("orbbec");
```

**HAL 推荐 B**——让用户 `apt install camera-hal-orbbec` 即可启用后端,不必重编 Adapter。

### 4.4 解码器插件

类似地,解码器通过 pluginlib 动态加载:
```xml
<library path="camera_hal_decoder_jpeg_turbo">
  <class type="camera_hal::JpegTurboDecoder"
         base_class_type="camera_hal::IDecoder"
         name="libjpeg-turbo"/>
</library>
```

工厂按平台优先级查找:
```cpp
auto& loader = IDecoderFactory::instance();
if (loader.hasDecoder("NV-Jetson")) return loader.create("NV-Jetson", MJPG);
if (loader.hasDecoder("RK-MPP"))    return loader.create("RK-MPP", MJPG);
return loader.create("libjpeg-turbo", MJPG);
```

### 4.5 Udev 规则

HAL 不应依赖特定 SDK 的 udev 规则。后端各自负责(Orbbec 后端安装 Orbbec udev,RealSense 后端依赖 librealsense2 包自带)。HAL 自己的 package 安装脚本仅做提示。

### 4.6 CI 构建矩阵

```yaml
# 推荐 CI 覆盖
matrix:
  os: [ubuntu-22.04, ubuntu-24.04]
  ros_distro: [humble, jazzy, rolling]
  backends:
    - orbbec
    - realsense
    - orbbec+realsense
  decoders:
    - jpeg-turbo
    - jpeg-turbo+jetson (只在 Jetson 自托管 runner 跑)
    - jpeg-turbo+rk-mpp (需要 ARM64 runner)
```

### 4.7 测试

- **单元测试**:用 Google Test;对 HAL 接口 mock(`MockIDevice`, `MockISensor`)
- **集成测试**:
  - 通过 `rosbag` 后端喂录制的数据,验证 Adapter 产出 topic
  - 需要硬件的测试标记为 `hardware_required`,CI 跳过
- **性能测试**:帧率、CPU 占用、内存,用 `test/perf_*.cpp`

### 4.8 坑

- ⚠️ **ABI 兼容**:SDK 升级可能破坏 ABI,后端 SO 与 HAL core SO 要版本对齐。建议 semver,core 大版本变动 → 重编所有后端。
- ⚠️ **编译时间长**:Orbbec 类单文件 220KB 编译慢(~30s);多文件拆分大幅改善。
- ⚠️ **CUDA 依赖**:Jetson decoder 需要 CUDA,CI runner 必须有。否则只能本地编。
- ⚠️ **ROS distro 差异**:Humble 用 ROS2 galactic API(`create_publisher<>`),Jazzy 略有变化。HAL Adapter 需加 `rclcpp::VERSION_MAJOR` 保护。
- ⚠️ **交叉编译 Jetson**:官方建议 Jetson 上直接 native 编译。交叉编译 CUDA 极其复杂,HAL 不应假设可交叉。
