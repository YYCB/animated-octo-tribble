# 安全与 OTA — 总索引

> **调研范围**：具身智能机器人的完整安全技术栈——从 ROS 2 DDS 通信层的 sros2/DDS-Security 加密认证，到节点级访问控制，再到 OTA 固件/软件安全更新。  
> **背景**：部署在公开环境中的机器人（医院、仓库、公共空间）面临通信截获、恶意节点注入、固件篡改等安全威胁，安全设计必须在架构阶段引入。

---

## 一、具身智能安全威胁模型

| 威胁类别 | 攻击向量 | 具体威胁 | 影响严重度 |
|---------|---------|---------|---------|
| **通信截获** | DDS 未加密话题 | 截获 `/camera/image_raw` 窃取视觉数据；截获 `/cmd_vel` 实施重放攻击 | 🔴 高 |
| **恶意节点注入** | 同一 ROS_DOMAIN_ID 网络 | 注入伪造 `/joint_commands`；伪造 `/odom` 干扰导航 | 🔴 高（安全事故） |
| **固件篡改** | 物理访问 / 网络 OTA | 篡改底盘 MCU 固件（micro-ROS）；替换 VLA 模型（对抗样本） | 🔴 高 |
| **物理安全** | 直接访问设备 | 提取 keystore；读取 rosbag2 录制的私密数据 | 🟡 中 |
| **DoS 攻击** | 大量 DDS 消息 | 淹没控制器消息队列；触发安全停机 | 🟡 中 |
| **供应链攻击** | 恶意依赖包 | 被污染的 apt/pip 包中含后门 | 🟡 中 |

│                                                                           │
│  节点认证层                                                                 │
│  ┌─────────────────────────────────────────────────────────────────────┐  │

│  │  MCU 固件 OTA（micro-ROS 层）                                        │  │
