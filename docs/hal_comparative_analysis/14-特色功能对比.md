# 14 — 特色功能对比 (LiDAR / Safety / Network)

> **本章回答**:两家相比彼此各有什么独特能力?这些能力要不要进 HAL?用什么形式进?

## 1. Orbbec 独有功能

### 1.1 LiDAR 支持(OBLidarNode)

**设备**:Pulsar ME450(固态 LiDAR)、Pulsar SL450(室外)。

**数据模型**:
```cpp
struct OBLiDARSpherePoint {
    float r;           // 距离(米)
    float theta;       // 水平角(度)
    float phi;         // 垂直角(度)
    float intensity;
};
```

**Publish Topic**:
- `/camera/scan` → `sensor_msgs/LaserScan`(2D 水平切片)
- `/camera/points` → `sensor_msgs/PointCloud2`(3D 笛卡尔)
- `/camera/sphere_points` → `sensor_msgs/PointCloud2`(3D 球坐标)
- `/camera/imu` → `sensor_msgs/Imu`(部分 LiDAR 内置 IMU)

**配置**:
```yaml
min_angle: -135.0
max_angle:  135.0
min_range:  0.05
max_range:  30.0
angle_increment:  0.25
```

### 1.2 网络设备一等公民

```yaml
enumerate_net_device: true
net_device_ip: "192.168.1.10"
net_device_port: 8090
```

SDK 提供 `ctx->createNetDevice(ip, port)`,设备对象等同 USB 设备。适用于:
- Femto Mega I(自带 RJ45)
- 远程部署(相机在远处,控制 PC 通过交换机访问)

### 1.3 Force IP 静态配置

通过 SDK 属性强制设置网络设备的 IP,防止 DHCP 漂移。

### 1.4 多设备同步 7 种模式

见 [第 9 章](09-多相机与硬件同步对比.md) § 1.3。

### 1.5 Interleave Frame(深度/IR 交替)

```yaml
enable_interleave_frame: true
```

用于交替开/关激光曝光,实现"有结构光 depth"+"无结构光 IR passive" 的帧交替。

### 1.6 Preset 系统

```yaml
device_preset: "High Accuracy"   # 或 "Default", "High Density", "Medium Density"
```

设备根据预设调整曝光/激光强度等,开箱即用的参数集。类似 RealSense 的 `json_file_path` 但粒度更粗、更用户友好。

### 1.7 D2C 双模式

```yaml
align_mode: "HW"   # 硬件对齐(芯片或 ISP 处理)
align_mode: "SW"   # 软件对齐(SDK 计算)
align_mode: "NONE"
```

### 1.8 进程锁

见 [第 2 章](02-设备发现与生命周期对比.md) §1.6。

### 1.9 固件升级

```yaml
upgrade_firmware: "/path/to/firmware.img"
```

Wrapper 内置流程,启动时自动升级。

## 2. RealSense 独有功能

### 2.1 安全相机(D585S)

**设备**:Intel RealSense D585S 功能安全相机(工业 SIL 认证)。

**特殊流类型**:
- `RS2_STREAM_LABELED_POINT_CLOUD`:带标签的点云
- `RS2_STREAM_OCCUPANCY`:占据栅格
- Safety depth stream

**特殊 Topic**:
```
/camera/labeled_pointcloud     # 带 voxel label 的 PointCloud2
/camera/occupancy              # 占据栅格图像
/camera/safety_log             # 安全日志
```

**特殊 Service**(6 个):
```
/camera/safety_preset_read         index → SafetyPreset
/camera/safety_preset_write        index, SafetyPreset →
/camera/safety_interface_config_read/write
/camera/serialized_device_config_read/write
```

**参数**:
```yaml
enable_safety: false
safety_camera.safety_mode: 0    # 0=Run, 1=Standby, 2=Service
```

### 2.2 Action API(TriggeredCalibration)

```
/camera/triggered_calibration    (Action)
  Goal: string json_config
  Result: bool success, float health
  Feedback: float progress, string status
```

长耗时(~10s)在线校准操作,异步,支持取消。

### 2.3 On-chip Calibration(D400)

- Tare Calibration:快速参考距离标定
- Self-Calibration:自动多角度标定

通过 `/camera/triggered_calibration` Action 调用。

### 2.4 Intra-process 双发布器

```
image_rcl_publisher            # rclcpp 原生,支持 zero-copy intra-process
image_transport_publisher      # 支持 compressed/theora 等 image_transport 插件
```

默认走 rcl;用户可切 image_transport。

### 2.5 GPU 加速(GLSL)

```yaml
accelerate_gpu_with_glsl: true
```

SDK 内部使用 OpenGL Compute / Fragment Shader 做:
- 颜色格式转换
- 视差→深度
- 点云生成

### 2.6 Rosbag 作为虚拟设备

```yaml
rosbag_filename: "/path/to/recording.bag"
rosbag_loop: true
```

SDK 把 bag 文件当成一种 device 打开,回放帧就像真实相机一样。这是 librealsense2 提供的特色。

### 2.7 RGBD 组合消息

`realsense2_camera_msgs::msg::RGBD` 把 color+depth+两套 camera_info 打包成一条消息,下游 SLAM 订阅一个 topic 即得对齐数据。

### 2.8 HDR Merge(序列化曝光)

硬件支持 HDR 的设备可以配多组曝光序列,Wrapper 通过 `depth_module.hdr_enabled` / `depth_module.exposure.1` / `depth_module.exposure.2` 控制。

### 2.9 Lifecycle 节点支持(编译期)

见 [第 1 章](01-架构与类层次对比.md) § 2.2。

### 2.10 Diagnostic 标准化

用 ROS 标准 `diagnostic_msgs::DiagnosticArray` 发到 `/diagnostics`,可被 rqt_runtime_monitor 等通用工具观察。

### 2.11 T265 / D555 DDS 相机

- T265:已停产但 SDK 仍支持(6DoF 追踪,pose stream)。
- D555:原生 DDS 相机,不通过 USB,直接发 ROS topics;设备本身有一个 RS2 节点。

## 4. 对 HAL 的决策

### 4.1 HAL 应该原生支持

以下能力显著提升 HAL 的价值,应该在 Core 接口里表达:

- ✅ **LiDAR**:作为一种 `SensorCategory::Lidar`,复用 `ISensor`/`IFrame`(frame data 是球坐标点)
- ✅ **Network Device**:`IDeviceManager::connectNetworkDevice(ip, port)` 已在 Core 接口
- ✅ **Sync Manager**:`ISyncManager` 以 Orbbec 7 模式为上界,RealSense 后端实现子集
- ✅ **On-chip Calibration / Firmware Update**:作为 `ProgressCallback` 长耗时接口,Adapter 层用 Action 暴露
- ✅ **Rosbag 后端**:作为 `IDeviceManager` 的一种特殊实例化(如 `create("rosbag", path)`),HAL 的 `IFrame` 语义天然支持

### 4.2 HAL 应该作为可选能力(capability query)

以下能力存在但不是所有设备/后端都支持,通过 `supportsXxx()` 查询:

- `IDevice::supportsFirmwareUpdate()`
- `IDevice::supportsAdvancedMode()`
- `IDevice::supportsOnChipCalibration()`
- `IDevice::supportsNetworkInterface()`
- `ISensor::supportsAutoExposureROI()`
- `ISyncManager::getSupportedSyncModes()`
- `IDevice::supportsSafetySensor()`
- `IFilter`:各种滤波器通过工厂查询

### 4.3 HAL 不原生支持,留给 Adapter 层

以下不进 HAL 核心,因为属于 ROS 生态的事:

- ❌ Lifecycle Node(ROS Adapter 层可选实现)
- ❌ Diagnostic(ROS Adapter 层发 `diagnostic_msgs`)
- ❌ Intra-process 发布器(ROS Adapter 层实现 `image_publisher` 双路)
- ❌ RGBD 组合消息(ROS Adapter 根据 config 决定)
- ❌ Preset YAML 模板(是 Launch 系统职责)

### 4.4 HAL 对特殊能力的扩展点

```cpp
// 通用的能力查询接口
class ICapabilityQuery {
public:
    virtual bool hasCapability(const std::string& name) const = 0;
    virtual std::vector<std::string> listCapabilities() const = 0;
};

class IDevice : public ICapabilityQuery { ... };

// 安全相机作为一种 ISensor 特化
class ISafetySensor : public ISensor {
public:
    enum class SafetyMode { Run, Standby, Service };
    virtual SafetyMode getSafetyMode() const = 0;
    virtual void setSafetyMode(SafetyMode) = 0;

    struct SafetyPreset { /* ... */ };
    virtual SafetyPreset readPreset(int index) = 0;
    virtual void writePreset(int index, const SafetyPreset&) = 0;
};

// Pose 流作为 IMotionFrame 的扩展(对 T265)
class IPoseFrame : public IFrame {
public:
    struct Pose {
        float translation[3];
        float quaternion[4];  // x, y, z, w
        float linear_velocity[3];
        float angular_velocity[3];
        float linear_acceleration[3];
        int tracker_confidence;
        int mapper_confidence;
    };
    virtual Pose getPose() const = 0;
};
```

### 4.5 可选扩展能力清单

HAL 接口中可选(Core 不强制,后端按需提供):

| 扩展 | 推荐接口 |
|---|---|
| 硬件对齐 | `AlignFilter::setHardwareAlign(bool)` |
| Interleave | `IDevice::setInterleaveEnabled(bool)` + `FrameSet` 含多帧 |
| Preset | `IDevice::loadPreset(std::string name)` + `listAvailablePresets()` |
| Force IP | `IDevice::setForceIPConfig(IPConfig)` |
| Label/Occupancy | 特殊 `StreamType::LabeledPointCloud` / `StreamType::Occupancy` |
| Pose Stream | 特殊 `StreamType::Pose6DoF`,数据是 `IPoseFrame` |

## 5. 对 HAL 实现者的建议

1. **Core 只收录两家共有的或明确有价值扩展的能力**,不要为了覆盖罕见能力污染接口。
2. **用能力查询替代硬判断**——`hasCapability("safety")` 比 `IDevice::supportsSafety()` 更扩展友好。
3. **LiDAR 不要做成单独业务节点**(Orbbec 犯的错),应作为 Sensor 子类型,业务逻辑在 Adapter 共用。
4. **Rosbag 后端尽早实现**——没有它,集成测试没法做。
5. **安全相机的 preset 是二进制 blob**——HAL 只传递字节流,不解析。
