# 10 — TF 与外参标定对比

> **本章回答**:相机内部各传感器(color/depth/IR/IMU)之间的外参怎么获取?怎么发布到 ROS TF?光学坐标系与 ROS 坐标系的转换约定是什么?HAL 的 `ICalibration` 接口该提供哪些数据?

---

## 1. Orbbec 侧:静态 TF + 外参 Topic

### 1.1 坐标系命名

```
{tf_prefix}{camera_name}_link              # 基准坐标系(可对外接驳到 robot URDF)
{tf_prefix}{camera_name}_color_frame
{tf_prefix}{camera_name}_color_optical_frame
{tf_prefix}{camera_name}_depth_frame
{tf_prefix}{camera_name}_depth_optical_frame
{tf_prefix}{camera_name}_ir_frame
{tf_prefix}{camera_name}_ir_optical_frame
{tf_prefix}{camera_name}_accel_frame
{tf_prefix}{camera_name}_gyro_frame
```

### 1.2 静态 TF 发布

`publishStaticTransforms()` 在 `OBCameraNode` 初始化末尾调用:
```
camera_link
  ├── color_frame
  │    └── color_optical_frame
  ├── depth_frame
  │    └── depth_optical_frame
  ├── ir_frame
  │    └── ir_optical_frame
  ├── accel_frame
  └── gyro_frame
```

外参从 SDK 获取:
```cpp
auto extrinsics = device->getCalibrationCameraParamList()
                       .extrinsicsForSpecificStream(OB_STREAM_DEPTH, OB_STREAM_COLOR);
// → 4x4 transform
```

### 1.3 ROS / Optical 坐标系约定

| | X | Y | Z |
|---|---|---|---|
| ROS Frame | 前 | 左 | 上 |
| Optical Frame | 右 | 下 | 前(光轴) |

变换:`Optical = ROS · R_optical`,其中 `R_optical` 把 ROS XYZ 映射到 Optical XYZ:
```
R_optical = [[0, 0, 1],
             [-1, 0, 0],
             [0, -1, 0]]
四元数: x=-0.5, y=0.5, z=-0.5, w=0.5
```

### 1.4 外参 Topic

除 TF 外,Orbbec 还把外参作为消息发布到 latched topic:
```
/camera/depth_to_color/extrinsics  → orbbec_camera_msgs::msg::Extrinsics
/camera/depth_to_ir/extrinsics    → orbbec_camera_msgs::msg::Extrinsics
```

```
# Extrinsics.msg
std_msgs/Header header
float64[9] rotation       # 3x3 行主序
float64[3] translation    # 米
```

### 1.5 CameraInfo

每个流发对应的 `CameraInfo` topic:
```
/camera/color/camera_info  → sensor_msgs/CameraInfo
/camera/depth/camera_info
/camera/ir/camera_info
```

包含:
- `K` (3x3 内参矩阵)
- `D` (5 项 plumb_bob 畸变 k1, k2, p1, p2, k3)
- `R` (3x3 修正矩阵,通常 identity)
- `P` (3x4 投影矩阵)
- `width / height`

### 1.6 用户标定

支持加载用户自定义标定:
```yaml
calibration_file_path: "/path/to/user_calib.yaml"
```

会覆盖出厂标定。Service 层提供:
- `/camera/get_user_calib_params`
- `/camera/set_user_calib_params`

### 1.7 IMU 标定

`getCalibrationCameraParamList()` 不直接给 IMU 标定。IMU 噪声/偏置参数通过参数文件提供。

## 2. RealSense 侧:静态 + 动态 TF + IMU Info

### 2.1 坐标系命名

```
{tf_prefix}{camera_name}_link              # base_frame_id
{tf_prefix}{camera_name}_depth_frame
{tf_prefix}{camera_name}_depth_optical_frame
{tf_prefix}{camera_name}_color_frame
{tf_prefix}{camera_name}_color_optical_frame
{tf_prefix}{camera_name}_infra1_frame
{tf_prefix}{camera_name}_infra1_optical_frame
{tf_prefix}{camera_name}_infra2_frame
{tf_prefix}{camera_name}_infra2_optical_frame
{tf_prefix}{camera_name}_gyro_frame
{tf_prefix}{camera_name}_gyro_optical_frame
{tf_prefix}{camera_name}_accel_frame
{tf_prefix}{camera_name}_accel_optical_frame
{tf_prefix}{camera_name}_imu_frame             # 软件融合 IMU
{tf_prefix}{camera_name}_imu_optical_frame
{tf_prefix}{camera_name}_aligned_depth_to_color_frame
```

### 2.2 TF 树

```
camera_link
  ├── camera_depth_frame
  │     └── camera_depth_optical_frame
  ├── camera_color_frame
  │     └── camera_color_optical_frame
  ├── camera_infra1_frame ─ ... 同
  ├── camera_infra2_frame ─ ...
  ├── camera_gyro_frame
  │     ├── camera_gyro_optical_frame
  │     └── camera_imu_frame  (当 imu_sync_method > NONE)
  │           └── camera_imu_optical_frame
  ├── camera_accel_frame
  │     └── camera_accel_optical_frame
  └── camera_aligned_depth_to_color_frame  (当 align_depth 启用)
```

### 2.3 静态 + 动态 TF

- **静态 TF**:`StaticTransformBroadcaster`,只发一次。
- **动态 TF**(可选):`tf_publish_rate > 0` 时,定时器周期性发布(用于设备热标定后更新外参)。

### 2.4 ROS / Optical 约定

完全相同(同 ROS REP-103 约定)。

### 2.5 外参获取

```cpp
auto from_profile = depth_stream_profile;
auto to_profile   = color_stream_profile;
rs2_extrinsics ext = from_profile.get_extrinsics_to(to_profile);
// ext.rotation: float[9], 列主序
// ext.translation: float[3]
```

注意:**RealSense 是列主序**,ROS msg 里通常按行主序填,要转置或注意约定一致。

### 2.6 外参消息

```
realsense2_camera_msgs::msg::Extrinsics
  float64[9] rotation       # 列主序 3x3
  float64[3] translation    # 米
```

发布到 `/camera/extrinsics/depth_to_color` 等,latched QoS。

### 2.7 IMUInfo Topic

```
realsense2_camera_msgs::msg::IMUInfo
  std_msgs/Header header
  float64[12] data             # 3x4 校准矩阵
  float64[3] noise_variances
  float64[3] bias_variances
```

发到 `/camera/accel/imu_info` 与 `/camera/gyro/imu_info`,latched。

### 2.8 在线校准

D400 设备支持 on-chip calibration(Tare、Self-Calibration),通过 `rs2::auto_calibrated_device::run_on_chip_calibration()` 调用。realsense-ros 通过 ROS Action 暴露:
```
/camera/run_on_chip_calibration  (Action)
  → 进度反馈,长耗时(~10s)
```

## 3. 对比总览

| 维度 | Orbbec | RealSense |
|---|---|---|
| 坐标系命名 | 一致(REP-103) | 一致 |
| Optical 转换 | 同(quaternion 0.5, -0.5, 0.5, -0.5) | 同 |
| 静态 TF | ✅ | ✅ |
| 动态 TF | ❌ | **✅** (`tf_publish_rate`) |
| Extrinsics msg | ✅(行主序) | **✅(列主序)** |
| Extrinsics topic | ✅ latched | ✅ latched |
| CameraInfo | ✅ | ✅ |
| IMUInfo topic | 通过 service | **✅ latched topic** |
| 用户标定 | ✅(yaml + service) | 部分(Tare) |
| 在线校准 | ❌ | **✅ Action**(D400) |
| Aligned depth frame | 通过 D2C 模式 | 独立的 `aligned_depth_to_color_frame` |
| `imu_optical_frame` | ❌ | **✅** |

### ⚠️ 关键差异

1. **行主序 vs 列主序**:
   - Orbbec Extrinsics msg 行主序。
   - RealSense Extrinsics msg 列主序。
   - **HAL 决策**:`Extrinsics` 内部用 **`Eigen::Matrix3d`** (列主序,默认 Eigen 行为),消息时统一行主序(更符合 ROS 习惯)。后端在转换时各自处理。

2. **动态 TF**:
   - 仅 RealSense 支持。
   - 用例:设备运行时被环境温度影响导致内参漂移,需要重新发外参。
   - **HAL 决策**:`ICalibration` 提供 `getCurrentExtrinsics()`,Adapter 决定是否周期发布。

3. **在线校准**:
   - 仅 RealSense D400 支持(SDK 提供 API)。
   - **HAL 决策**:可选能力 `IDevice::supportsOnChipCalibration()`,通过 Action 实现长耗时操作。

## 4. 对 HAL 的可复用要点

### 4.1 数据结构

```cpp
struct Intrinsics {
    int width;
    int height;
    float fx, fy;            // 焦距
    float ppx, ppy;          // 主点
    enum DistortionModel {
        None,
        BrownConrady,        // OpenCV plumb_bob (Orbbec 默认)
        InverseBrownConrady, // RealSense 用于 color
        Kannala_Brandt4,     // 鱼眼
    } distortion_model;
    float distortion_coeffs[5];  // k1, k2, p1, p2, k3
};

struct Extrinsics {
    // 标准化为列主序 Eigen 矩阵,ROS 转换时由 Adapter 处理
    Eigen::Matrix3f rotation;
    Eigen::Vector3f translation;  // 米
};
```

### 4.2 `ICalibration` 接口

```cpp
class ICalibration {
public:
    virtual ~ICalibration() = default;

    // 内参
    virtual Intrinsics getIntrinsics(const StreamProfile&) = 0;

    // 外参(任意两流之间)
    virtual Extrinsics getExtrinsics(const StreamProfile& from,
                                      const StreamProfile& to) = 0;

    // 深度元数据
    virtual float getDepthScale() = 0;          // 米/单位 (D435 ≈ 0.001)
    virtual float getDepthMin() = 0;             // 最小有效深度(米)
    virtual float getDepthMax() = 0;

    // IMU
    virtual IMUCalibration getIMUCalibration(StreamType type) = 0;

    // 用户标定加载
    virtual bool supportsUserCalibration() = 0;
    virtual void loadUserCalibration(const std::string& yaml_path) = 0;

    // 在线校准(可选)
    virtual bool supportsOnChipCalibration() = 0;
    using ProgressCallback = std::function<void(float progress, const std::string& status)>;
    virtual void runOnChipCalibration(const std::string& json_config,
                                      ProgressCallback cb) = 0;
};

class IDevice {
    virtual std::shared_ptr<ICalibration> getCalibration() = 0;
};
```

### 4.3 ROS Adapter 的 TF 发布逻辑

```cpp
void publishStaticTransforms(std::shared_ptr<ICalibration> calib,
                              const std::vector<StreamProfile>& active) {
    StreamProfile base = pickBaseStream(active);  // 通常是 depth

    // 1. base_link → base_stream_frame (identity)
    publishStatic(base_frame_id, frameId(base), Eigen::Matrix4f::Identity());

    // 2. base_stream_frame → base_stream_optical_frame (固定旋转)
    publishStatic(frameId(base), opticalFrameId(base), kRosToOpticalRotation);

    // 3. 其他每个 stream:
    //    base_stream_frame → other_stream_frame  (从 calib 取外参)
    //    other_stream_frame → other_stream_optical_frame (固定旋转)
    for (auto& other : active) {
        if (other == base) continue;
        auto ext = calib->getExtrinsics(base, other);
        publishStatic(frameId(base), frameId(other), eigenFrom(ext));
        publishStatic(frameId(other), opticalFrameId(other), kRosToOpticalRotation);
    }

    // 4. IMU 特殊处理(如启用)
    if (gyro_active) {
        auto ext = calib->getExtrinsics(base, gyro_profile);
        publishStatic(frameId(base), "gyro_frame", ext);
        publishStatic("gyro_frame", "gyro_optical_frame", kRosToOpticalRotation);
        // 软件融合时:imu_frame = gyro_frame
    }
}
```

### 4.4 动态 TF(可选周期发)

```cpp
if (tf_publish_rate > 0) {
```

### 4.5 CameraInfo 构建

```cpp
sensor_msgs::msg::CameraInfo intrinsicsToCameraInfo(const Intrinsics& I) {
    sensor_msgs::msg::CameraInfo info;
    info.width  = I.width;
    info.height = I.height;
    info.distortion_model = (I.distortion_model == Intrinsics::BrownConrady) ? "plumb_bob" : "...";
    info.d = std::vector<double>(I.distortion_coeffs, I.distortion_coeffs + 5);
    info.k = {I.fx, 0,    I.ppx,
              0,    I.fy, I.ppy,
              0,    0,    1};
    info.r = {1,0,0, 0,1,0, 0,0,1};   // identity (单目)
    info.p = {I.fx, 0,    I.ppx, 0,
              0,    I.fy, I.ppy, 0,
              0,    0,    1,     0};
    return info;
}
```


### 4.7 坑

- ⚠️ **行/列主序混乱**:HAL 内部统一一种(推荐 Eigen 默认列主序),Adapter 在写入 ROS msg 时显式 transpose。
- ⚠️ **Optical 旋转的四元数符号**:不同实现可能给出 (0.5, -0.5, 0.5, -0.5) 或 (-0.5, 0.5, -0.5, 0.5),数值上等价但 RViz 可能报警。HAL 提供常量。
- ⚠️ **CameraInfo 的 P 矩阵**:对单目 P[:,:3] 等于 K, P[:,3] = 0;对 stereo 还要填 baseline*fx。HAL 不必做 stereo,但要文档说明。
- ⚠️ **深度单位**:D435 是 mm(0.001 m/unit),Orbbec 不同设备不同(有些是 0.1mm)。`getDepthScale` 是必需的,Adapter 不能假设 0.001。
