## 1. Orbbec 侧:12 级深度后处理 + 多变体

### 1.1 滤波器顺序(`setupDepthPostProcessFilter`)

```
原始深度帧
   │
   ▼ ① DecimationFilter            降采样(scale: 2/4/8)
   ▼ ② HDRMerge                    HDR 合成(多曝光合并)
   ▼ ③ SequenceIdFilter            多曝光模式序列选择
   ▼ ④ ThresholdFilter             阈值过滤(min/max)
   ▼ ⑤ HardwareNoiseRemovalFilter  硬件噪声去除
   ▼ ⑥ NoiseRemovalFilter          软件噪声去除(min_diff=256, max_size=80)
   ▼ ⑦ SpatialFilter               空间平滑(alpha, diff, magnitude, radius)
   ▼ ⑧ SpatialFastFilter           快速空间滤波变体
   ▼ ⑨ SpatialModerateFilter       中等空间滤波变体
   ▼ ⑩ TemporalFilter              时间平滑(diff_threshold, weight)
   ▼ ⑪ HoleFillingFilter           孔洞填充(nearest / farthest 等)
   ▼ ⑫ DisparityToDepth            视差转深度
   │
   ▼
处理后的深度帧
```

### 1.2 Spatial 三种变体

| 滤波器 | 速度 | 质量 | 适用 |
|---|---|---|---|
| `SpatialFilter` | 慢 | 高 | 离线/高端设备 |
| `SpatialFastFilter` | 快 | 中 | 实时/嵌入式 |
| `SpatialModerateFilter` | 中 | 中高 | 折中 |

Orbbec 让用户显式选其一(通过 enable_xxx 参数),而不是自动切换。

### 1.3 Hardware vs Software Noise Removal

`HardwareNoiseRemovalFilter` 是 SDK/固件提供的,速度更快但可调性有限。`NoiseRemovalFilter` 是软件版,参数更丰富(`min_diff` / `max_size`)。可以同时启用。

### 1.4 IR 帧的独立滤波链

```
processLeftIrFrameFilter(frame):
    SequenceIdFilter → DecimationFilter

processRightIrFrameFilter(frame):
    SequenceIdFilter → DecimationFilter
```

IR 滤波器链很短,因为 IR 数据不像深度那样需要复杂后处理。

### 1.5 滤波状态 JSON 发布

```cpp
nlohmann::json filter_status_;
filter_status_["spatial"]["enable"] = enable_spatial_filter_;
filter_status_["spatial"]["alpha"]  = spatial_filter_alpha_;
filter_status_["temporal"]["weight"] = temporal_filter_weight_;
// ...
filter_status_pub_->publish(filter_status_.dump());
```

用户可以订阅 `/camera/filter_status` topic 查看当前滤波器配置(调试友好)。

### 1.6 深度对齐(D2C)

**两档**:

```cpp
// HW 模式:硬件完成
if (align_mode_ == "HW") {
    pipeline_config_->setAlignMode(ALIGN_D2C_HW_MODE);
}
// SW 模式:SDK 软件完成
else if (align_mode_ == "SW") {
    align_filter_ = std::make_unique<ob::Align>(OB_STREAM_COLOR);
    auto aligned = align_filter_->process(frame_set);
}
```

### 1.7 图像翻转/镜像/旋转(非 SDK 滤波)

Wrapper 在 `onNewFrameCallback` 里用 OpenCV 做:
```cpp
if (mirror_stream_[sip]) cv::flip(image, image, 1);   // 水平镜像
if (flip_stream_[sip])   cv::flip(image, image, 0);   // 垂直翻转
if (rotation_stream_[sip] > 0) cv::rotate(image, image, rotation_code);
```

### 1.8 畸变矫正(针对彩色)

```cpp
if (enable_color_undistortion_) {
    cv::undistort(image, result, K, D, new_K);
}
```

## 2. RealSense 侧:规范化 10+ 级 + NamedFilter 基类

### 2.1 `NamedFilter` 基类设计

```cpp
class NamedFilter {
protected:
    std::shared_ptr<rs2::filter> _filter;
    SensorParams _params;
    rclcpp::Logger _logger;
    bool _is_enabled;
    std::vector<std::string> _parameters_names;

public:
    NamedFilter(shared_ptr<rs2::filter> f, shared_ptr<Parameters> p, Logger l,
                bool is_enabled = false, bool register_options = true);

    rs2::frameset process(rs2::frameset frameset);
    bool is_enabled() const;
    void setParameters(std::function<void(const Parameter&)> enable_cb);
};
```

### 2.2 参数自动注册(亮点)

```cpp
void NamedFilter::setParameters(enable_callback) {
    // 获取 SDK 滤波器名称 (如 "Decimation Filter" → "decimation_filter")
    std::string module_name = rs2_to_ros(_filter->get_info(RS2_CAMERA_INFO_NAME));

    // 注册使能参数: {filter_name}.enable
    std::string param_name = module_name + ".enable";
    _params.getParameters()->setParamT(param_name, _is_enabled,
        [this, enable_callback](const Parameter& p) {
            _is_enabled = p.get_value<bool>();
            if (enable_callback) enable_callback(p);
        });

    // 注册其他 option 参数 (如 decimation_filter.magnitude, spatial_filter.alpha)
    _params.registerDynamicOptions(*_filter, module_name);
}
```

**效果**:SDK 新增滤波器或给现有滤波器加参数,Wrapper **零代码改动**。

### 2.3 `setupFilters()` 滤波管线

```cpp
void BaseRealSenseNode::setupFilters() {
    _filters.push_back(NamedFilter(make_shared<rs2::decimation_filter>()));        // 1. 抽取
    _filters.push_back(NamedFilter(make_shared<rs2::hdr_merge>()));                // 2. HDR 合并
    _filters.push_back(NamedFilter(make_shared<rs2::sequence_id_filter>()));       // 3. 序列 ID
    _filters.push_back(NamedFilter(make_shared<rs2::disparity_transform>()));      // 4. 深度→视差
    _filters.push_back(NamedFilter(make_shared<rs2::spatial_filter>()));           // 5. 空间滤波
    _filters.push_back(NamedFilter(make_shared<rs2::temporal_filter>()));          // 6. 时间滤波
    _filters.push_back(NamedFilter(make_shared<rs2::hole_filling_filter>()));      // 7. 孔洞填充
    _filters.push_back(NamedFilter(make_shared<rs2::disparity_transform>(false))); // 8. 视差→深度
    _filters.push_back(NamedFilter(make_shared<rs2::rotation_filter>()));          // 9. 旋转

    // 特殊滤波器
    _pc_filter    = PointcloudFilter(make_shared<rs2::pointcloud>());              // 10. 点云
    _align_filter = AlignDepthFilter(make_shared<rs2::align>(...));                // 11. 深度对齐
    _colorizer    = NamedFilter(make_shared<rs2::colorizer>());                    // 12. 深度着色
}
```

### 2.4 子类化的两个特殊滤波器

```
NamedFilter
  ├── PointcloudFilter     : 点云生成 + 自己做发布
  └── AlignDepthFilter     : 深度对齐 + aligned_depth Topic
```

**为什么要子类化**:
- 点云/对齐需要**直接使用 ROS Publisher**(把 `PointCloud2` 发出去),纯 `rs2::filter` 不够。
- `isWantedProfile`/ `getQOS` 需要特化。

### 2.5 处理循环(在 frame_callback)

```cpp
for (auto& filter : _filters) {
    if (filter->is_enabled())
        frameset = filter->process(frameset);
}
```

**按注册顺序应用**——顺序由 SDK 推荐决定(参考 librealsense Documentation "post-processing filters")。

### 2.6 深度对齐:一种滤波器

`AlignDepthFilter` 本质是 `rs2::align(RS2_STREAM_COLOR)` 的包装,放在 filter 列表中作为一级。启用时生成一个 aligned depth 帧(保留原 depth 帧的同时)。

### 2.7 GPU 加速

```
accelerate_gpu_with_glsl = true
  └── SDK 内部 glsl_colorizer / glsl_spatial_filter 等 GLSL 实现
```

Wrapper 无感,只是开关 SDK 能力。

## 3. 对比总览

| 维度 | Orbbec | RealSense |
|---|---|---|
| 滤波器总数 | **12 级**(深度) + IR 链 + 对齐 + 翻转 | **10+** 级 + Pointcloud + Align + Colorizer |
| 基类抽象 | ❌(散放在 OBCameraNode 成员) | **`NamedFilter`** 基类 + 2 子类 |
| 参数注册 | **手工**逐个声明 | **自动**(`registerDynamicOptions`) |
| Spatial 变体 | **3 种**(standard / fast / moderate) | 1 种(靠参数调) |
| 噪声去除 | HW + SW 双版本 | 无独立"noise_removal" filter |
| HDR | 滤波器链内(`HDRMerge` + `SequenceIdFilter`) | 同样,且还有 RosSensor 级 HDR 序列管理 |
| 深度对齐 | HW / SW 两档(参数选择) | 仅作为 `AlignDepthFilter` 一级 |
| 点云 | Wrapper 侧生成 | `PointcloudFilter` 一级 |
| 翻转/镜像/旋转 | Wrapper 侧 cv::flip | `rotation_filter`(SDK 内置) |
| GPU 加速 | ❌(靠解码器 HW) | `accelerate_gpu_with_glsl` |
| 状态查询 | 发 `filter_status` JSON topic | ROS 参数本身反映状态 |

### ⚠️ 关键差异

1. **抽象强度**:
   - RealSense 的 `NamedFilter` 基类 + 两个子类 = 漂亮的 OO 设计,HAL 应直接借鉴。
   - Orbbec 每个 filter 的 enable 位和参数都散放在 `OBCameraNode` 成员里,维护性差。

2. **参数**:
   - RealSense 自动注册 filter 的所有 SDK option 为 ROS 参数,这是 [第 5 章](05-参数系统对比.md) 的延伸。
   - Orbbec 手工注册,每个 filter 十几个参数累计上百个。

3. **深度对齐定位**:
   - Orbbec 把对齐当作"Pipeline 配置"(HW) 或"独立 filter"(SW)。
   - RealSense 纯粹把它当"滤波器链的一级"。
   - **HAL 决策**:统一为 `IFilter` 的一种。硬件对齐作为后端内部优化(当 HW 模式可用时,SDK 直接生成对齐帧,不走 filter 调用)。

4. **顺序的可配置性**:
   - 两家都是硬编码顺序(SDK 推荐)。
   - **HAL 决策**:`IFilterPipeline` 应允许用户按需增删和重排,默认顺序预置。

## 4. 对 HAL 的可复用要点

### 4.1 `IFilter` 接口

```cpp
class IFilter {
public:
    virtual ~IFilter() = default;

    virtual std::string getName() const = 0;
    virtual bool isEnabled() const = 0;
    virtual void setEnabled(bool) = 0;

    // 处理单帧
    virtual std::shared_ptr<IFrame> process(std::shared_ptr<IFrame>) = 0;

    // 处理 frameset(默认实现:对每帧调 process;子类可重写为跨帧操作)
    virtual std::shared_ptr<IFrameSet> process(std::shared_ptr<IFrameSet>) = 0;

    // 选项(复用 OptionInfo,见第 5 章)
    virtual std::vector<OptionInfo> getSupportedOptions() const = 0;
    virtual float getOption(const std::string& name) const = 0;
    virtual void setOption(const std::string& name, float value) = 0;

    // 可选:目标流类型过滤(只对 depth 起作用?还是 color?)
    virtual std::vector<StreamType> getApplicableStreams() const {
        return { StreamType::Depth };  // default
    }
};
```

### 4.2 `IFilterPipeline` 接口

```cpp
class IFilterPipeline {
public:
    virtual ~IFilterPipeline() = default;

    // 管线管理
    virtual void addFilter(std::shared_ptr<IFilter>) = 0;
    virtual void addFilter(std::shared_ptr<IFilter>, size_t position) = 0;
    virtual void removeFilter(const std::string& name) = 0;
    virtual std::vector<std::shared_ptr<IFilter>> getFilters() const = 0;
    virtual std::shared_ptr<IFilter> getFilter(const std::string& name) const = 0;

    // 处理
    virtual std::shared_ptr<IFrameSet> process(std::shared_ptr<IFrameSet>) = 0;

    // 默认构造(基于后端推荐的滤波器与顺序)
    static std::shared_ptr<IFilterPipeline> createDefault(std::shared_ptr<IDevice>);
};
```

### 4.3 两个特化子类(类比 RealSense)

```cpp
class PointCloudFilter : public IFilter {
public:
    // 除了 process(depth frameset) 产出点云帧外,
    // 还持有 RGB 映射配置(有彩色则生成 XYZRGB)
    virtual void setColorSource(StreamIndex color_stream) = 0;

    // 输出通过 process 的返回的 frameset 里的 StreamType::PointCloud 获取
};

class AlignFilter : public IFilter {
public:
    // 对齐到哪个目标流
    virtual void setAlignTarget(StreamIndex target) = 0;
    // 后端可选硬件对齐 / 软件对齐
    virtual bool supportsHardwareAlign() const = 0;
    virtual void setHardwareAlign(bool) = 0;
};
```

### 4.4 自动参数注册(ROS Adapter 层)

```cpp
void registerFilterAsRosParams(std::shared_ptr<IFilter> filter,
                                std::shared_ptr<IParameterProvider> provider) {
    std::string prefix = filter->getName();  // "decimation_filter"

    // enable
    provider->declareParameter(prefix + ".enable",
        filter->isEnabled(),
        [filter](bool v){ filter->setEnabled(v); });

    // options
    for (auto& opt : filter->getSupportedOptions()) {
        std::string p = prefix + "." + opt.name;
        provider->declareParameter(p, opt, filter->getOption(opt.name),
            [filter, opt](float v){ filter->setOption(opt.name, v); },
            [filter, opt](){ return filter->getOption(opt.name); });
    }
}
```

### 4.6 Filter 状态查询

两种方式,HAL 可都支持:
1. **ROS 参数本身即状态**(RealSense 模式,推荐)
2. **独立 `/filter_status` JSON topic**(Orbbec 模式,辅助调试)

Adapter 层实现 (1) 即可满足主要用户。

### 4.7 翻转/镜像/旋转的归属

- 两家处理方式不同:Orbbec 在 callback 里用 cv::flip,RealSense 有 SDK `rotation_filter`。
- **HAL 决策**:作为 HAL 内置的 **`FlipMirrorRotateFilter`**,默认注册但不启用,由 ROS 参数控制。不要像 Orbbec 那样把这段 OpenCV 散写在 callback 里。

### 4.8 畸变矫正的归属

- 也作为 HAL 的 `UndistortFilter`,独立于 `AlignFilter`。
- 参数:每流独立的 `undistort.enable` / `undistort.target_camera_matrix`。

### 4.9 坑

- ⚠️ **滤波顺序敏感**:Depth ↔ Disparity 必须成对,Temporal 通常在 Spatial 后。错序会导致输出错误。HAL 默认顺序应与 SDK 官方推荐一致。
- ⚠️ **Filter 是有状态的**:Temporal/HoleFilling 有历史帧缓存。参数变化时需要重置状态(两家 SDK 都通过 `set_option` 隐式重置)。HAL 应在文档中明确 `setOption` 可能清空滤波器内部状态。
- ⚠️ **`process` 不能原地修改输入**:返回新 frame 对象(语义上),SDK 内部可能引用计数复用,但用户不能假设。
- ⚠️ **Point cloud filter 性能**:CPU 版点云在 D435 848x480 下会占用数百 ms/帧,必须能关闭或转 GPU 版本。
